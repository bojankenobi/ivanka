1. Uvod u Objektno Orijentisano Programiranje (OOP)

Dobrodošli na predavanje o Objektno Orijentisanom Programiranju! Danas ćemo zakoračiti u svet modernog programiranja i naučiti kako da razmišljamo o problemima na nov, intuitivniji način. Ovaj pristup nam pomaže da pišemo organizovaniji, fleksibilniji i lakši za održavanje kod, što je ključno za razvoj kompleksnih softverskih sistema.

Glavni cilj OOP-a je upravljanje složenošću (complexity). Kako softver raste, postaje sve teže pratiti sve delove i njihove međusobne veze. OOP nam daje alate da grupišemo povezane podatke i funkcionalnosti u samostalne jedinice (objekte), što drastično smanjuje mentalno opterećenje i omogućava timovima da efikasnije sarađuju.

Proceduralno Programiranje

Zamislite da pravite sendvič prateći recept korak po korak. To je suština proceduralnog programiranja. Pišemo niz instrukcija (procedura ili funkcija) koje računar izvršava redom. Fokus je na akcijama.

  • Fokus na procedurama i funkcijama.
  • Podaci i funkcije su odvojeni.
  • Teže za održavanje kod velikih projekata.

Objektno Orijentisano Programiranje (OOP)

Sada zamislite da gradite kuću od LEGO kockica. Svaka kockica je "objekat" sa svojim osobinama (boja, veličina) i mogućnostima (može se spojiti sa drugom). U OOP-u, svet posmatramo kroz objekte koji međusobno komuniciraju. Fokus je na stvarima (objektima).

  • Fokus na objektima koji objedinjuju podatke i ponašanje.
  • Kod je modularan i lakši za ponovnu upotrebu.
  • Intuitivnije preslikava stvarni svet u program.

2. Stubovi OOP-a: Pregled

Objektno orijentisano programiranje se zasniva na nekoliko ključnih principa koji ga čine moćnim. Mi ćemo ih zvati "stubovima" OOP-a. Razumevanje ovih koncepata je temelj za efikasno korišćenje OOP paradigme. Oni nam omogućavaju da gradimo softver koji je robustan, prilagodljiv i lak za razumevanje.

Klase i Objekti: Nacrt i Kuća

Ovo je najfundamentalniji koncept u OOP-u. Sve počinje odavde. Moramo jasno razlikovati šta je plan (klasa), a šta je stvarna stvar napravljena po tom planu (objekat).

Klasa (Class)

Klasa je nacrt ili šablon za kreiranje objekata. Ona definiše:

  • Atribute (Attributes): Osobine koje će objekat imati (npr. boja, ime, brzina). To su promenljive unutar klase.
  • Metode (Methods): Akcije koje će objekat moći da izvršava (npr. vozi, koči, laj). To su funkcije unutar klase.

Na primer, klasa `Automobil` bi definisala da svaki automobil ima boju, marku i brzinu, i da može da ubrza i zakoči.

Objekat (Object)

Objekat je instanca (konkretan primerak) klase. To je stvarna "stvar" kreirana prema nacrtu klase, sa konkretnim vrednostima za svoje atribute.

  • Moj plavi "Audi" je jedan objekat.
  • Tvoj crveni "BMW" je drugi objekat.
  • Oba su objekti klase `Automobil`.

Objekat je ono što "živi" u memoriji računara i sa čime radimo u programu.

Primer u Javi

Hajde da vidimo kako izgleda jednostavna klasa `Ucenik` u Javi.

public class Ucenik {
// Atributi
String ime;
String prezime;
int razred;

// Metoda
public void predstaviSe() {
System.out.println("Zdravo, ja sam " + ime + " " + prezime + " i idem u " + razred + ". razred.");
}
}

Sada, hajde da kreiramo dva objekta (instance) te klase:

// Kreiranje prvog objekta
Ucenik ucenik1 = new Ucenik();
ucenik1.ime = "Petar";
ucenik1.prezime = "Petrović";
ucenik1.razred = 3;

// Kreiranje drugog objekta
Ucenik ucenik2 = new Ucenik();
ucenik2.ime = "Jovana";
ucenik2.prezime = "Jovanović";
ucenik2.razred = 3;

// Pozivanje metode nad objektima
ucenik1.predstaviSe();
ucenik2.predstaviSe();

Konstruktori i `this` ključna reč

Kada kreiramo objekat sa `new Ucenik()`, mi zapravo pozivamo specijalnu metodu koja se zove konstruktor. Njegov zadatak je da inicijalizuje početno stanje objekta.

  • Svaka klasa ima podrazumevani (default) konstruktor ako ga mi ne definišemo.
  • Možemo definisati i sopstvene konstruktore da bismo primili početne vrednosti za atribute.
  • Ključna reč `this` se odnosi na trenutni objekat. Koristi se za razlikovanje atributa klase od parametara metode (ili konstruktora) kada imaju isto ime.
public class Ucenik {
String ime;
String prezime;
int razred;

// Naš konstruktor koji prihvata početne vrednosti
public Ucenik(String ime, String prezime, int razred) {
// 'this.ime' je atribut klase
// 'ime' je parametar prosleđen konstruktoru
this.ime = ime;
this.prezime = prezime;
this.razred = razred;
}

public void predstaviSe() {
System.out.println("Zdravo, ja sam " + this.ime + " " + this.prezime);
}
}

// Sada je kreiranje objekata mnogo čistije:
Ucenik ucenik1 = new Ucenik("Petar", "Petrović", 3);
Ucenik ucenik2 = new Ucenik("Jovana", "Jovanović", 3);
ucenik1.predstaviSe();

Enkapsulacija: Sigurnosna Kutija

Enkapsulacija je princip "pakovanja" podataka (atributa) i metoda koje rade sa tim podacima u jednu celinu - objekat. Ali to nije sve! Ključni deo je sakrivanje informacija (information hiding).

Zamislite bankomat. Vi možete da ubacite karticu, unesete PIN i podignete novac. Ne znate (i ne treba da znate) kako bankomat interno proverava stanje na računu ili broji novčanice. On vam nudi jednostavan interfejs, a sakriva unutrašnju složenost. To je enkapsulacija.

Kako to radi?

  • Atributi klase se deklarišu kao private. To znači da im se ne može direktno pristupiti van klase.
  • Kreiraju se javne metode, tzv. geteri i seteri (public), koje služe kao kontrolisani pristup privatnim atributima.

Zašto je ovo važno?

  • Kontrola: Seteri nam omogućavaju da dodamo logiku za proveru pre nego što promenimo vrednost atributa (npr. ocena ne može biti veća od 5).
  • Sigurnost: Sprečava se slučajna ili namerna izmena stanja objekta na nepredviđen način.
  • Fleksibilnost: Možemo promeniti unutrašnju implementaciju klase bez da utičemo na kod koji je koristi, sve dok javni interfejs (metode) ostaje isti.
  • Održivost: Kada je unutrašnje stanje objekta zaštićeno, lakše je pronaći i ispraviti greške, jer znamo da se stanje moglo promeniti samo kroz definisane setere.

Primer: Klasa `Ucenik` sa enkapsulacijom

public class Ucenik {
// Atributi su sada private!
private String ime;
private int ocena;

// Javni seter za 'ime'
public void setIme(String novoIme) {
this.ime = novoIme;
}

// Javni geter za 'ime'
public String getIme() {
return this.ime;
}

// Seter za 'ocena' sa logikom za proveru
public void setOcena(int novaOcena) {
if (novaOcena >= 1 && novaOcena <= 5) {
this.ocena = novaOcena;
} else {
System.out.println("Greška: Ocena mora biti između 1 i 5.");
}
}

// Geter za 'ocena'
public int getOcena() {
return this.ocena;
}
}

Sada, umesto direktnog pristupa `ucenik1.ocena = 10;`, moramo koristiti metodu:

Ucenik ucenik1 = new Ucenik();
ucenik1.setIme("Marko");
ucenik1.setOcena(5); // Ovo je u redu
ucenik1.setOcena(7); // Ovo će ispisati grešku!

System.out.println("Učenik: " + ucenik1.getIme() + ", Ocena: " + ucenik1.getOcena());

Nasleđivanje: Porodično Stablo

Nasleđivanje (Inheritance) je mehanizam koji omogućava jednoj klasi da "pokupi" (nasledi) atribute i metode od druge klase. Ovo je jedan od najmoćnijih načina za ponovnu upotrebu koda u OOP-u.

Ključni termini

  • Nadklasa (Superclass): Klasa od koje se nasleđuje. Naziva se i roditeljska (parent) ili osnovna (base) klasa.
  • Podklasa (Subclass): Klasa koja nasleđuje. Naziva se i dečija (child) ili izvedena (derived) klasa.

Koristimo ključnu reč extends da bismo ostvarili nasleđivanje.

Primer iz života

Imamo opšti koncept `Vozilo`. Svako vozilo ima brzinu i može da se kreće.

`Automobil` je specifična vrsta vozila. On nasleđuje sve od `Vozila` (ima brzinu, kreće se), ali dodaje i nešto svoje, npr. broj vrata.

`Bicikl` je takođe vrsta vozila. I on nasleđuje od `Vozila`, ali ima svoj specifičan atribut, npr. broj brzina.

"IS-A" Odnos (JESTE)

Nasleđivanje modelira "IS-A" (jeste) odnos. Ovo je ključno za pravilno korišćenje. Pre nego što upotrebite nasleđivanje, postavite sebi pitanje: Da li je podklasa zaista vrsta nadklase?
✔ `Pas` jeste `Životinja`. (Dobro)
✔ `Automobil` jeste `Vozilo`. (Dobro)
❌ `Automobil` nije `Točak`. (Loše - ovo bi bio "HAS-A" odnos)

Primer u Javi

Prvo, osnovna (roditeljska) klasa:

public class Zivotinja {
String ime;

public void jedi() {
System.out.println("Ova životinja jede.");
}
}

Sada, izvedena (dečija) klasa koja nasleđuje `Zivotinju`:

// Pas JE Životinja
public class Pas extends Zivotinja {
public void laj() {
System.out.println("Av av!");
}
}

// Macka JE Životinja
public class Macka extends Zivotinja {
public void mjauci() {
System.out.println("Mjau!");
}
}

Kako se koristi:

Pas mojPas = new Pas();
mojPas.ime = "Lesi"; // 'ime' je nasleđeno iz klase Zivotinja
mojPas.jedi(); // 'jedi()' je nasleđeno iz klase Zivotinja
mojPas.laj(); // 'laj()' je specifično za klasu Pas

Ključna reč `super`

Ključna reč `super` se koristi unutar podklase da bi se pozvale metode ili konstruktori iz nadklase. Ovo je korisno kada želimo da proširimo, a ne da potpuno zamenimo, ponašanje nadklase.

public class Osoba {
String ime;
public Osoba(String ime) {
this.ime = ime;
System.out.println("Kreirana je Osoba.");
}
public void predstaviSe() {
System.out.println("Ja sam osoba, zovem se " + ime);
}
}

public class Student extends Osoba {
String fakultet;
public Student(String ime, String fakultet) {
// 1. Pozivamo konstruktor nadklase (Osoba)
super(ime);
this.fakultet = fakultet;
System.out.println("Kreiran je Student.");
}

@Override
public void predstaviSe() {
// 2. Pozivamo metodu predstaviSe() iz nadklase
super.predstaviSe();
System.out.println("Studiram na " + fakultet);
}
}

// Upotreba:
Student s = new Student("Milan", "ETF");
s.predstaviSe();

Primer 1: Svet Geometrijskih Oblika (Princip Specijalizacije)

Konceptualna Analiza: Od Opšteg ka Specifičnom

Jedan od najintuitivnijih primera za razumevanje nasleđivanja potiče iz geometrije. Zamislimo hijerarhiju oblika. Možemo reći da Kvadrat jeste Pravougaonik, ali sa jednim dodatnim, specifičnim pravilom: sve stranice su mu jednake. Dalje, Pravougaonik jeste Oblik. Ova hijerarhija je savršen primer principa specijalizacije. Krećemo od veoma opšteg koncepta (Oblik) i postepeno ga činimo specifičnijim.

Dizajn Klasa: Planiranje Naše Hijerarhije

  • Oblik (Nadklasa): Biće najopštija klasa. Sadržaće samo jedno zajedničko polje za sve oblike: boja.
  • Pravougaonik (Izvedena iz Oblik): Naslediće polje boja i dodati svoja specifična polja: sirina i visina. Takođe će imati metodu za izračunavanje površine, povrsina().
  • Kvadrat (Izvedena iz Pravougaonik): Naslediće sve od Pravougaonika. Njegov konstruktor će biti specijalizovan tako da prima samo dužinu jedne stranice i postavlja da su i širina i visina jednake toj vrednosti.

Implementacija u Javi

// Nadklasa (roditelj)
class Oblik {
String boja;

public Oblik(String boja) {
this.boja = boja;
}
}

// Podklasa (dete) koja nasleđuje klasu Oblik
class Pravougaonik extends Oblik {
double sirina;
double visina;

public Pravougaonik(String boja, double sirina, double visina) {
// Pozivamo konstruktor nadklase (Oblik) da bi on postavio polje 'boja'.
// super() mora biti prva linija koda u konstruktoru podklase.
super(boja);
this.sirina = sirina;
this.visina = visina;
}

public double povrsina() {
return this.sirina * this.visina;
}
}

// Podklasa koja nasleđuje klasu Pravougaonik
class Kvadrat extends Pravougaonik {
public Kvadrat(String boja, double stranica) {
// Pozivamo konstruktor nadklase (Pravougaonik)
// i prosleđujemo 'stranica' za širinu i visinu.
super(boja, stranica, stranica);
}
}

// Glavna klasa za testiranje
public class GeometrijaTest {
public static void main(String[] args) {
Pravougaonik p = new Pravougaonik("Plava", 10, 5);
System.out.println("Boja pravougaonika: " + p.boja);
System.out.println("Površina pravougaonika: " + p.povrsina());

System.out.println();

Kvadrat k = new Kvadrat("Crvena", 7);
System.out.println("Boja kvadrata: " + k.boja);
// Kvadrat može da koristi metodu povrsina() iako nije definisana u klasi Kvadrat!
System.out.println("Površina kvadrata: " + k.povrsina());
}
}

Analiza Koda: Ključni Mehanizmi Nasleđivanja

Ključna reč extends: Linija class Pravougaonik extends Oblik govori Javi da klasa Pravougaonik nasleđuje sva public i protected polja i metode od klase Oblik.

Poziv super(): Poziv super(boja) je izuzetno važan. To je način na koji podklasa poziva konstruktor svog direktnog roditelja. U klasi Pravougaonik, super(boja) osigurava da se deo koda iz klase Oblik koji postavlja boju izvrši. Bez ovoga, polje boja ne bi bilo inicijalizovano, a program se ne bi kompajlirao.

Nasleđivanje metoda: Iako klasa Kvadrat nema sopstvenu metodu povrsina(), objekat k je može bez problema pozvati. To je zato što je Kvadrat nasledio ovu metodu od svog roditelja, klase Pravougaonik. Ovo je suština ponovne upotrebe koda.

Ključni Uvid Primera 1

Nasleđivanje omogućava specijalizaciju — proces uzimanja opšteg koncepta (kao što je Pravougaonik) i kreiranja njegove specifičnije verzije (Kvadrat) koja zadržava sve osobine opšteg, ali dodaje ili modifikuje ponašanje.

Primer 2: Klasifikacija u Životinjskom Carstvu (Princip Proširenja)

Konceptualna Analiza: Građenje Složenosti

Biološka klasifikacija je još jedan savršen primer hijerarhijske strukture. Možemo reći da Pas jeste Sisar, a Sisar jeste Zivotinja. Ovde se fokusiramo na proširenje (extension). Svaki niži nivo u hijerarhiji dodaje nove sposobnosti i atribute. Zivotinja se hrani. Sisar se hrani i doji. Pas se hrani, doji i laje. Svaki nivo nasleđivanja proširuje klasu novim funkcionalnostima.

Dizajn Klasa

  • Zivotinja: Osnovna klasa sa poljem starost i metodom jedi().
  • Sisar (od Zivotinja): Nasleđuje sve od Zivotinja, ali dodaje novo polje bojaKrzna i novu metodu doji().
  • Pas (od Sisar): Nasleđuje sve od Sisar, ali dodaje novo polje rasa i novu metodu laj().

Implementacija u Javi

class Zivotinja {
int starost;

public Zivotinja(int starost) {
this.starost = starost;
}

public void jedi() {
System.out.println("Ova životinja jede.");
}
}

class Sisar extends Zivotinja {
String bojaKrzna;

public Sisar(int starost, String bojaKrzna) {
super(starost);
this.bojaKrzna = bojaKrzna;
}

public void doji() {
System.out.println("Ovaj sisar doji svoje mladunce.");
}
}

class Pas extends Sisar {
String rasa;

public Pas(int starost, String bojaKrzna, String rasa) {
super(starost, bojaKrzna);
this.rasa = rasa;
}

public void laj() {
System.out.println("Av! Av!");
}
}

public class ZivotinjeTest {
public static void main(String[] args) {
Pas lesi = new Pas(5, "zlatna", "Škotski ovčar");

// Pristup poljima sa svih nivoa hijerarhije
System.out.println("Lesi je star " + lesi.starost + " godina."); // Iz klase Zivotinja
System.out.println("Boja krzna: " + lesi.bojaKrzna); // Iz klase Sisar
System.out.println("Rasa: " + lesi.rasa); // Iz klase Pas

System.out.println();

// Poziv metoda sa svih nivoa hijerarhije
lesi.jedi(); // Metoda nasleđena od Zivotinja
lesi.doji(); // Metoda nasleđena od Sisar
lesi.laj(); // Metoda iz klase Pas
}
}

Analiza Koda

Ovaj primer savršeno ilustruje kako objekat najniže klase u hijerarhiji, Pas, akumulira sva polja i metode svojih predaka. Objekat lesi ima pristup poljima starost (iz Zivotinja), bojaKrzna (iz Sisar) i rasa (iz Pas). Ovo je primer višeslojnog nasleđivanja (multilevel inheritance).

Posebno je zanimljiv "lanac konstrukcije". Kada kreiramo objekat Pas, njegov konstruktor poziva super(...), što je konstruktor klase Sisar. Zatim, konstruktor klase Sisar poziva super(...), što je konstruktor klase Zivotinja. Na ovaj način, svaka klasa u lancu nasleđivanja obavlja svoj deo posla u inicijalizaciji objekta.

Ključni Uvid Primera 2

Nasleđivanje se koristi za proširenje funkcionalnosti. Izvedene klase nisu samo specijalizovane verzije svojih roditelja; one su često i bogatije i sposobnije, jer nadograđuju postojeću osnovu novim poljima i metodama.

Primer 3: Hijerarhija Vozila (Princip Grupisanja Zajedničkih Osobina)

Konceptualna Analiza: Pronalaženje Sličnosti

Sada ćemo pogledati situaciju gde više različitih klasa nasleđuje jednu zajedničku baznu klasu. Zamislimo različita vozila. Automobil jeste Vozilo i Bicikl takođe jeste Vozilo. Svrha ovog primera je da pokaže kako se zajedničke osobine grupišu u roditeljskoj klasi da bi se izbeglo ponavljanje koda. Na primer, i automobil i bicikl imaju maksimalnu brzinu i broj točkova.

Ovaj tip nasleđivanja, gde više podklasa nasleđuje jednu nadklasu, naziva se hijerarhijsko nasleđivanje.

Implementacija u Javi

class Vozilo {
int maksimalnaBrzina;
int brojTockova;

public Vozilo(int maksimalnaBrzina, int brojTockova) {
this.maksimalnaBrzina = maksimalnaBrzina;
this.brojTockova = brojTockova;
}
}

class Automobil extends Vozilo {
int brojVrata;

public Automobil(int maksimalnaBrzina, int brojVrata) {
// Automobil uvek ima 4 točka, pa fiksiramo tu vrednost
super(maksimalnaBrzina, 4);
this.brojVrata = brojVrata;
}
}

class Bicikl extends Vozilo {
String tipRama;

public Bicikl(int maksimalnaBrzina, String tipRama) {
// Bicikl uvek ima 2 točka
super(maksimalnaBrzina, 2);
this.tipRama = tipRama;
}
}

public class VozilaTest {
public static void main(String[] args) {
Automobil mojAuto = new Automobil(220, 5);
System.out.println("--- Automobil ---");
System.out.println("Maksimalna brzina: " + mojAuto.maksimalnaBrzina + " km/h");
System.out.println("Broj točkova: " + mojAuto.brojTockova); // Nasleđeno od Vozilo
System.out.println("Broj vrata: " + mojAuto.brojVrata);

System.out.println();

Bicikl mojBicikl = new Bicikl(40, "aluminijumski");
System.out.println("--- Bicikl ---");
System.out.println("Maksimalna brzina: " + mojBicikl.maksimalnaBrzina + " km/h");
System.out.println("Broj točkova: " + mojBicikl.brojTockova); // Nasleđeno od Vozilo
System.out.println("Tip rama: " + mojBicikl.tipRama);
}

import java.util.Arrays;

double plata;

this.id = id;

}

Ključni Uvid Primera 4

Nasleđivanje u kombinaciji sa redefinisanjem metoda omogućava podklasama da pruže specifičnu implementaciju za ponašanja koja su generalno definisana u nadklasi. Ovo je osnova za polimorfizam, jedan od najmoćnijih koncepata u OOP-u.

Polimorfizam: Više Oblika

Polimorfizam na grčkom znači "imati više oblika". U Javi, polimorfizam nam omogućava da izvršimo istu akciju na različite načine. Postoje dva glavna tipa polimorfizma:

1. Statički (Compile-time) Polimorfizam: Preopterećenje Metoda (Method Overloading)

Ovo se dešava kada u istoj klasi imamo više metoda sa istim imenom, ali različitim parametrima (različit broj parametara, različit tip, ili oboje). Kompajler odlučuje koju metodu da pozove na osnovu argumenata koje smo prosledili.

class Kalkulator {
public int saberi(int a, int b) {
return a + b;
}

// Ista metoda, ali sa 3 parametra
public int saberi(int a, int b, int c) {
return a + b + c;
}

// Ista metoda, ali sa double tipom
public double saberi(double a, double b) {
return a + b;
}
}

Kalkulator k = new Kalkulator();
System.out.println(k.saberi(2, 3)); // Poziva prvu metodu
System.out.println(k.saberi(2, 3, 4)); // Poziva drugu metodu
System.out.println(k.saberi(2.5, 3.5)); // Poziva treću metodu

2. Dinamički (Run-time) Polimorfizam: Redefinisanje Metoda (Method Overriding)

Ovo se dešava kada podklasa pruži specifičnu implementaciju metode koja je već definisana u njenoj nadklasi. Odluka o tome koja metoda će se izvršiti donosi se u vreme izvršavanja programa (run-time), na osnovu tipa objekta.

Ovo je ono što najčešće podrazumevamo pod "polimorfizmom" u OOP kontekstu. Ključno je za fleksibilnost i nadogradivost koda.

class Zivotinja {
public void oglasiSe() {
System.out.println("Životinja se oglašava");
}
}

class Pas extends Zivotinja {
@Override
public void oglasiSe() {
System.out.println("Pas laje: Av av!");
}
}

// Upotreba
Zivotinja z = new Pas(); // z je tipa Zivotinja, ali objekat je Pas
z.oglasiSe(); // U vreme izvršavanja, JVM vidi da je objekat Pas
// i poziva metodu iz klase Pas.

Apstrakcija: Fokusiranje na Bitno

Apstrakcija je proces sakrivanja implementacionih detalja i prikazivanja samo funkcionalnosti korisniku. Drugim rečima, korisnik zna šta objekat radi, ali ne i kako to radi.

Dobar primer je vožnja automobila. Da biste vozili, morate znati da koristite volan, pedale i menjač (interfejs). Ne morate znati kako motor sa unutrašnjim sagorevanjem radi, kako se snaga prenosi na točkove, ili kako radi elektronski sistem ubrizgavanja goriva (implementacija).

Kako se postiže u Javi?

Apstrakcija se u Javi najčešće postiže korišćenjem:

  • Apstraktnih klasa (Abstract Classes): Klasa koja se ne može instancirati (ne može se kreirati objekat od nje). Može imati i apstraktne (bez implementacije) i konkretne metode. Služi kao osnova za druge klase koje je nasleđuju.
  • Interfejsa (Interfaces): Potpuno apstraktan "ugovor" koji definiše skup metoda koje klasa mora da implementira. Ne može imati implementaciju metoda (u starijim verzijama Jave).

Zašto je ovo važno?

  • Smanjuje složenost: Fokusiramo se na interakciju sa objektom na višem nivou.
  • Olakšava održavanje: Možemo promeniti internu implementaciju bez promene spoljnog koda koji je koristi.
  • Povećava sigurnost: Sprečava korisnika da napravi promene koje bi mogle da naruše konzistentnost objekta.

Primer: Apstraktna klasa `Oblik`

Definišemo apstraktnu klasu `Oblik`. Svaki oblik mora imati mogućnost da izračuna svoju površinu, ali ne znamo kako se to radi za "apstraktni" oblik. Zato je metoda `povrsina()` apstraktna, i svaka konkretna klasa koja nasledi `Oblik` mora da pruži svoju implementaciju.

// Apstraktna klasa - ne može se kreirati objekat tipa Oblik
abstract class Oblik {
// Apstraktna metoda - nema telo, samo potpis
// Svaka podklasa MORA da je implementira
public abstract double povrsina();

// Apstraktna klasa može imati i konkretne metode
public void prikazi() {
System.out.println("Ovo je neki geometrijski oblik.");
}
}

class Krug extends Oblik {
private double poluprecnik = 5.0;

// Pružamo implementaciju za apstraktnu metodu
@Override
public double povrsina() {
return Math.PI * poluprecnik * poluprecnik;
}
}

// Greška! Ne možemo kreirati objekat apstraktne klase.
// Oblik o = new Oblik();

// Ovo je ispravno
Oblik krug = new Krug();
krug.prikazi();
System.out.println("Površina kruga je: " + krug.povrsina());

Unified Modeling Language (UML) je standardizovani grafički jezik za vizuelizaciju, specificiranje, konstruisanje i dokumentovanje softverskih sistema. To je kao arhitektonski nacrt za zgradu, samo što je za softver. Pomaže nam da planiramo i komuniciramo ideje pre nego što napišemo kod.

Postoji mnogo vrsta UML dijagrama, ali za nas je na početku najvažniji Dijagram Klasa (Class Diagram), jer direktno prikazuje strukturu našeg OOP koda.

Dijagram klasa prikazuje klase u sistemu, njihove atribute, metode i odnose između klasa. Osnovni element je pravougaonik podeljen na tri dela:

  • Srednji deo: Lista atributa.

Lekcija 1: Šta je baza podataka i čemu služi?

Uvod: Problem neorganizovanih podataka

Zamislimo da smo bibliotekari u školskoj biblioteci i da sve podatke vodimo u jednoj velikoj svesci. Svaki put kada neko pozajmi knjigu, mi to zapišemo. Posle nekog vremena, naša sveska izgleda ovako:

Ovo na prvi pogled radi, ali šta se dešava ako želimo da odgovorimo na neka pitanja?

  • Pitanje: Koje sve knjige je pozajmio Petar Petrović? Problem: Moramo ručno da prelistamo celu svesku i tražimo njegovo ime. Ako je sveska dugačka, to traje.

  • Pitanje: Kako da unesemo novu knjigu koju smo upravo kupili, a koju još niko nije pozajmio? Problem: U našoj svesci nema mesta za to. Ona prati samo iznajmljivanja.

Ovaj haos neorganizovanih podataka je problem koji rešavaju baze podataka.

Korak 1: Od sveske do organizovanih tabela (Entiteti)

Umesto jedne velike, neuredne sveske, hajde da razmislimo pametnije. Koje su glavne "stvari" o kojima mi čuvamo podatke? To su učenici i knjige. U svetu baza podataka, ove "stvari" nazivamo entiteti. Hajde da napravimo dve odvojene, čiste liste (tabele), po jednu za svaki entitet.

Dali smo svakom učeniku i svakoj knjizi jedinstveni identifikacioni broj (ID). Ovo je izuzetno važno, jer sada možemo imati dva učenika sa istim imenom, ali će oni imati različit ID.

Korak 2: Definicija Baze Podataka (BP)

Baza podataka (BP) je jednostavno kolekcija ovih organizovanih i međusobno povezanih tabela. To nije jedna tabela, već ceo sistem tabela koje zajedno čine smislenu celinu.

Korak 3: Upoznavanje "Bibliotekara" (DBMS)

Imamo organizovane tabele, ali nam treba nešto što će upravljati njima. Program koji će nam omogućiti da dodajemo nove učenike, pretražujemo knjige i, što je najvažnije, da pazi na pravila. Taj program se zove Sistem za Upravljanje Bazama Podataka (DBMS - Database Management System). Najlakše je zamisliti ga kao pametnog bibliotekara za naše podatke. DBMS je softver koji mi koristimo da bismo radili sa bazom. Najpoznatiji DBMS sistemi su Microsoft SQL Server, MySQL, Oracle i PostgreSQL.

Zaključka lekcije

  • Podaci su sirove, neorganizovane činjenice (haotična sveska).
  • Informacije su organizovani podaci koji imaju smisao (uredne tabele).
  • Baza podataka (BP) je kolekcija organizovanih tabela.
  • Sistem za Upravljanje Bazama Podataka (DBMS) je softver (bibliotekar) koji koristimo da upravljamo bazom, unosimo podatke i osiguravamo da se poštuju pravila.

Lekcija 2: Vrste Baza Podataka - SQL vs. NoSQL

Uvod: Dva načina za organizaciju podataka

U prošloj lekciji smo videli kako podatke organizujemo u uredne tabele. Taj pristup se zove relacioni. Međutim, sa pojavom interneta, pojavila se potreba za fleksibilnijim pristupom. Danas imamo dve velike "porodice" baza podataka: SQL (relacione) i NoSQL (nerelacione). Hajde da ih objasnimo kroz analogiju: kako biste organizovali svoju odeću?

Primer 1: SQL Baza Podataka = Savršeno organizovan ormar

Zamislimo ormar sa jasno definisanim pregradama. Postoje stroga pravila (šema), struktura je predvidiva, a pouzdanost visoka. Jezik za rad sa ovim bazama je SQL (Structured Query Language). SQL baze su idealne za podatke koji imaju jasnu, nepromenljivu strukturu (npr. podaci o učenicima, bankovni računi).

Primer 2: NoSQL Baza Podataka = Kutija za sve i svašta

Zamislimo veliku kutiju u koju odlažete razne stvari. Fleksibilnost je ključna (bez fiksne šeme). Ne postoje fiksne tabele; podaci se mogu čuvati kao dokumenti ili ključ-vrednost parovi. NoSQL baze su dizajnirane za brzinu i laku skalabilnost (dodavanje novih "kutija"). Idealne su za raznovrsne i nestrukturirane podatke (npr. postovi na društvenim mrežama, komentari).

Korak po korak: Upoređivanje

KarakteristikaSQL (Ormar)NoSQL (Kutija)

Šema (Pravila)Fiksna, mora se definisati unapredFleksibilna, može se menjati u hodu
SkalabilnostTeže se skalira (kupuje se veći ormar)Lako se skalira (dodaje se još kutija)

Zaključka lekcije

Ne postoji "bolja" ili "lošija" baza podataka. Postoji samo pravi alat za pravi posao. U nastavku kursa, fokusiraćemo se na SQL baze, jer su one temelj i osnova.

Lekcija 3: Kako isplanirati bazu podataka? (Modeliranje)

Uvod: Pravljenje nacrta pre gradnje

Proces planiranja strukture baze zove se dizajn ili modeliranje. Dobar plan će nam uštedeti mnogo muka kasnije.

Korak 1: Pronalaženje glavnih "stvari" (Entiteti)

Prvo pitanje je: "O čemu želimo da čuvamo podatke?" Za našu biblioteku, entiteti su: Učenici, Knjige, i sam čin iznajmljivanja, dakle Iznajmljivanja.

Korak 2: Opisivanje "stvari" (Atributi)

Sada za svaki entitet definišemo osobine (atribute). Za Učenika to su Ime, Prezime, Razred. Za Knjigu su Naslov, Autor. Za Iznajmljivanje su DatumIznajmljivanja i DatumVracanja.

Korak 3: Rešavanje problema jedinstvenosti (Primarni ključ)

Šta ako imamo dva učenika sa istim imenom? Rešenje je jedinstveni identifikacioni broj, koji se zove Primarni ključ (Primary Key - PK). Svaka tabela (Ucenici, Knjige, Iznajmljivanja) dobiće svoju ID kolonu (UcenikID, KnjigaID, IznajmljivanjeID).

Korak 4: Povezivanje tabela (Spoljni ključ)

Kako da znamo koji učenik je pozajmio koju knjigu? Koristimo ID-jeve! U tabelu Iznajmljivanja nećemo upisati ime učenika, već samo njegov UcenikID. Kolona u jednoj tabeli koja pokazuje na primarni ključ u drugoj tabeli zove se Spoljni ključ (Foreign Key - FK). On je "lepak" koji povezuje naše tabele.

Finalni nacrt (Model) naše baze

  • Tabela: Ucenici (UcenikID (PK), Ime, Prezime, Razred)
  • Tabela: Knjige (KnjigaID (PK), Naslov, Autor)
  • Tabela: Iznajmljivanja (IznajmljivanjeID (PK), UcenikID (FK), KnjigaID (FK), DatumIznajmljivanja, DatumVracanja)

Lekcija 4: Gradimo bazu! (Kreiranje tabela i unos podataka)

Uvod: Od nacrta do gradilišta

Sada ćemo naš nacrt pretvoriti u pravu bazu pomoću SQL-a (Structured Query Language), jezika za komunikaciju sa bazama podataka.

Korak 1: Kreiranje tabela komandom CREATE TABLE

Njome kažemo DBMS-u: "Napravi mi novu, praznu tabelu sa ovim kolonama".

CREATE TABLE Ucenici (
    UcenikID INT PRIMARY KEY,
    Ime VARCHAR(50) NOT NULL,
    Prezime VARCHAR(50) NOT NULL,
    Razred VARCHAR(10)
);

Raščlanjeno: CREATE TABLE Ucenici (napravi tabelu), UcenikID INT PRIMARY KEY (kolona UcenikID, tipa ceo broj, je primarni ključ), Ime VARCHAR(50) NOT NULL (kolona Ime, tekst do 50 karaktera, ne sme biti prazna).

Korak 2: Unos podataka komandom INSERT INTO

Sada imamo prazne tabele. Vreme je da ih popunimo podacima.

INSERT INTO Ucenici (UcenikID, Ime, Prezime, Razred) VALUES (1, 'Petar', 'Petrović', 'IV-2');

Korak 3: Evidentiranje iznajmljivanja

Evidentiraćemo da je Petar Petrović (ID=1) pozajmio "Anu Karenjinu" (ID=101).

INSERT INTO Iznajmljivanja (IznajmljivanjeID, UcenikID, KnjigaID, DatumIznajmljivanja)
VALUES (1, 1, 101, '2025-10-01');

Obratite pažnju kako nismo pisali imena, već samo ID brojeve. Baza sada "zna" tačno ko je uzeo koju knjigu. Da smo probali da unesemo nepostojeći UcenikID, DBMS bi javio grešku, čuvajući tako integritet podataka.

Zaključka lekcije

Naučili smo osnovne SQL komande: CREATE TABLE za pravljenje strukture i INSERT INTO za unos podataka.

Lekcija 5: Postavljanje pitanja bazi (Upiti i forme)

Uvod: Izvlačenje informacija iz podataka

Prava moć baza podataka leži u mogućnosti da im postavljamo kompleksna pitanja. Taj proces se zove pisanje upita.

Korak 1: Osnovni upit komandom SELECT

Najvažnija komanda u SQL-u je SELECT. Njome kažemo: "Prikaži mi ove podatke". Zvezdica (*) znači "sve kolone". Za uslove koristimo klauzulu WHERE.

SELECT Ime, Prezime FROM Ucenici WHERE Razred = 'IV-2';

Korak 2: Povezivanje tabela u upitu (JOIN)

Postavićemo pitanje: "Prikaži mi imena učenika i naslove knjiga koje su pozajmili." Za ovo nam trebaju podaci iz više tabela, pa ih spajamo komandom JOIN.

SELECT Ucenici.Ime, Knjige.Naslov
FROM Iznajmljivanja
INNER JOIN Ucenici ON Iznajmljivanja.UcenikID = Ucenici.UcenikID
INNER JOIN Knjige ON Iznajmljivanja.KnjigaID = Knjige.KnjigaID;

JOIN je kao rajsferšlus koji spaja dve tabele koristeći zajedničke ključeve (ID-jeve).

Korak 3: Zašto su nam potrebne forme?

Pisanje SQL upita je moćno, ali nepraktično za krajnje korisnike. Zato za njih pravimo forme (obrasce) - grafički interfejs sa poljima i dugmićima koji olakšava rad sa bazom. Kada korisnik na formi klikne "Sačuvaj", program u pozadini sam sastavi i izvrši odgovarajuću SQL komandu. Forme čine bazu podataka upotrebljivom za obične korisnike, skrivajući složenost SQL-a "ispod haube".

Sekcija 1: Uvod u ApexSQL Refactor: Vaš Asistent za Profesionalni SQL Kod

U svetu programiranja, a posebno u radu sa bazama podataka, često se susrećemo sa kodom koji "radi posao". On izvršava zadatak, vraća podatke i naizgled ispunjava svoju svrhu. Međutim, postoji ogromna razlika između koda koji samo radi i koda koji je dobar. Dobar kod nije samo funkcionalan; on je čitljiv, razumljiv, efikasan i, što je najvažnije, lak za održavanje i nadogradnju. Tu na scenu stupa koncept refaktorisanja.

Šta je refaktorisanje i zašto bi vas to zanimalo?

Najjednostavnije rečeno, refaktorisanje je proces restrukturiranja postojećeg kompjuterskog koda – menjanje njegove unutrašnje strukture bez promene spoljašnjeg ponašanja. Zamislite to kao sređivanje i reorganizaciju vaše sobe. Sve vaše stvari su i dalje tu i služe istoj svrsi, ali su sada uredno složene, logično raspoređene i lako dostupne. Kada je soba sređena, mnogo je lakše pronaći ono što vam treba i dodati nove stvari bez stvaranja haosa.

Isto važi i za kod. Refaktorisanjem se ne dodaje nova funkcionalnost, već se poboljšava postojeća. Cilj je da kod postane:

  • Čitljiviji: Drugi programeri (ili vi sami, šest meseci kasnije) mogu brzo da razumeju šta kod radi.
  • Jednostavniji: Smanjuje se nepotrebna kompleksnost, što olakšava pronalaženje i ispravljanje grešaka.
  • Lakši za održavanje: Kada dođe vreme za promenu ili nadogradnju, to se može uraditi brže i sa manjim rizikom od uvođenja novih grešaka.

Usvajanje navike redovnog refaktorisanja je jedan od ključnih koraka koji deli amaterskog programera od profesionalca. To je disciplina koja pokazuje poštovanje prema kodu, kolegama i budućem sebi.

Zašto je formatiranje važno? Više od pukog izgleda.

Mnogi početnici potcenjuju važnost formatiranja, smatrajući ga čisto estetskom kategorijom. Međutim, njegov uticaj je mnogo dublji i direktno utiče na kvalitet i cenu razvoja softvera.

  • Smanjenje kognitivnog opterećenja: Ljudski mozak ima ograničen kapacitet radne memorije. Kada čitate neuredan kod – sa nedoslednim uvlačenjem, pomešanim velikim i malim slovima i bez logičkih preloma – vaš mozak troši dragocene resurse samo na "dešifrovanje" strukture, umesto da se fokusira na logiku. Dobro formatiran kod je vizuelno organizovan i omogućava vam da trenutno prepoznate ključne delove upita (SELECT, FROM, WHERE, JOIN), oslobađajući mentalni kapacitet za rešavanje stvarnih problema.
  • Uspostavljanje timskih standarda: U profesionalnom okruženju, retko kada radite sami. Kod koji vi napišete danas, sutra će možda morati da čita, razume i menja vaš kolega. Ako svako u timu formatira kod na svoj način, nastaje konfuzija i usporava se rad. Uspostavljanje i automatsko sprovođenje jedinstvenog stila formatiranja je ključno za efikasan timski rad.

ApexSQL Refactor vam omogućava da definišete i primenite ova pravila jednim klikom, pretvarajući haos u red.

Korak po korak: Podešavanje vašeg stila

  • Formatting: Ovde podešavate osnovnu strukturu – da li koristite tabulatore ili razmake za uvlačenje, koliko razmaka, da li želite prazne linije pre ili posle određenih naredbi, i da li se identifikatori stavljaju u uglaste zagrade ([]).
  • Capitalization: Jedna od najvidljivijih promena. Ovde definišete da li će ključne reči (SELECT, FROM, CREATE), tipovi podataka (INT, NVARCHAR), promenljive i funkcije biti pisane velikim ili malim slovima. Preporuka je da ključne reči uvek budu velikim slovima radi lakšeg vizuelnog razlikovanja.

Najbolji deo ovog prozora je Preview panel sa desne strane. Dok menjate bilo koju od stotina dostupnih opcija, odmah vidite kako će ta promena uticati na primer koda. Ovo stvara izuzetno brzu i efikasnu petlju učenja. Možete slobodno eksperimentisati sa svim podešavanjima bez ikakvog rizika.

Praktičan primer: Od haosa do reda

Zamislimo da ste napisali upit koji spaja podatke o učenicima i njihovim ocenama. U žurbi, niste obraćali pažnju na formatiranje.

Kod PRE:

select u.ime, p.nazivpredmeta, o.ocena from dbo.ucenici as u join dbo.ocene as o on u.ucenikid = o.ucenikid join dbo.predmeti p on o.predmetid = p.predmetid where o.ocena > 3 and p.nazivpredmeta like 'Matematika%' order by u.ime;

Ovaj kod je funkcionalan, ali je noćna mora za čitanje. Teško je na prvi pogled razdvojiti kolone, tabele, uslove spajanja i filtriranja.

Akcija: Sada, hajde da primenimo magiju. Selektujte ceo blok koda u SSMS-u. Zatim, koristite prečicu na tastaturi: Ctrl + Shift + Alt + F.

Kod POSLE:

SELECT
U.Ime,
P.NazivPredmeta,
O.Ocena
FROM
dbo.Ucenici AS U
JOIN dbo.Ocene AS O
ON U.UcenikID = O.UcenikID
JOIN dbo.Predmeti AS P
ON O.PredmetID = P.PredmetID
WHERE
O.Ocena > 3
AND P.NazivPredmeta LIKE 'Matematika%'
ORDER BY
U.Ime;

Razlika je drastična. Odmah možemo da vidimo:

  • Velika slova za ključne reči: SELECT, FROM, JOIN, WHERE se jasno ističu.
  • Svaka klauzula u novom redu: Logički delovi upita su vizuelno odvojeni.
  • Kolone na posebnim linijama: Lista kolona je pregledna.
  • Pravilno uvlačenje: JOIN uslovi su uvučeni, što jasno pokazuje hijerarhiju i strukturu upita.

Sekcija 3: Primer 2 (Početni nivo): Unapređenje Upita - Od Zvezdice (*) do Eksplicitnih Kolona

Jedna od prvih stvari koju naučimo kada počinjemo sa SQL-om je prečica SELECT *. Ona je brza, laka za kucanje i vraća nam sve kolone iz tabele. Za brzu proveru podataka tokom razvoja, ona je sasvim u redu. Međutim, korišćenje SELECT * u produkcionom kodu – kodu koji se izvršava u aplikacijama, izveštajima ili procedurama – smatra se jednom od najgorih praksi, i to sa dobrim razlogom.

Zašto je zvezdica (*) opasna? Skriveni problemi SELECT *.

Na prvi pogled, SELECT * deluje bezazleno. Ipak, iza te jednostavnosti kriju se ozbiljni rizici po performanse i stabilnost vašeg sistema.

  • Performanse: Kada napišete SELECT *, vi terate SQL Server da uradi dve nepotrebne stvari. Prvo, on mora da konsultuje sistemske tabele da bi saznao koja sve imena kolona postoje. Drugo, i važnije, vi skoro uvek prenosite više podataka nego što je vašoj aplikaciji zaista potrebno. Ako tabela Ucenici ima 20 kolona, a vama trebaju samo Ime i Prezime, SELECT * će nepotrebno opteretiti i bazu podataka i mrežni saobraćaj.
  • Stabilnost koda (Robusnost): Ovo je najkritičniji problem. Vaš kod i vaša baza podataka imaju nevidljivi "ugovor". Kada napišete SELECT *, taj ugovor je vrlo labav. Zamislite da vaša aplikacija očekuje da treća kolona bude DatumRodjenja. Šta se dešava ako neko doda novu kolonu u tabelu koja se umeće pre datuma rođenja? Vaša aplikacija će se srušiti. Korišćenje SELECT * čini vaš kod izuzetno krhkim.

Eksplicitnim navođenjem imena kolona, vi kreirate čvrst i jasan ugovor: "Potrebne su mi tačno ove kolone: Ime, Prezime, DatumRodjenja". Sada, ako neko doda novu kolonu, vaš kod će nastaviti da radi bez problema. Ovo pretvara Wildcard expansion iz obične stilske prepravke u moćan alat za proaktivno sprečavanje budućih grešaka.

Korak po korak: Proširivanje džokera (Wildcard Expansion)

ApexSQL Refactor čini ispravljanje ove loše prakse trivijalno lakim. Zamislimo da imamo sledeći upit u našem editoru:

SELECT * FROM dbo.Ucenici;

Da biste ovo refaktorisali, jednostavno pozicionirajte kursor miša bilo gde unutar zvezdice (*), desni klik, i izaberite ApexSQL Refactor -> Wildcard expansion.

Praktičan primer: Od implicitnog ka eksplicitnom

Kod PRE:

SELECT
*
FROM
dbo.Ucenici;

Kod POSLE:

SELECT
U.UcenikID,
U.Ime,
U.Prezime,
U.DatumRodjenja,
U.Prosek
FROM
dbo.Ucenici AS U;

Obratite pažnju na nekoliko pametnih stvari koje je alat uradio automatski:

  • Zamenio je * sa eksplicitnom listom svih kolona.
  • Dodao je alijas U tabeli Ucenici.
  • Prefiksovao je svako ime kolone sa alijasom tabele (U.Ime, U.Prezime, itd.).

Dodatni savet: Kvalifikacija imena objekata

Prefiksovanje imena kolona alijasom tabele (ili punim imenom tabele) naziva se kvalifikacija imena. Ovo je još jedna dobra praksa koju ApexSQL Refactor može da automatizuje pomoću funkcije Qualify object names.

Zašto je ovo važno?

  • Čitljivost u složenim upitima: Kada imate upit sa više JOIN operacija, vrlo je verovatno da će se ista imena kolona pojaviti u više tabela (npr. Ime u tabeli Ucenici i Ime u tabeli Profesori). Bez kvalifikacije, SQL Server će prijaviti grešku "Ambiguous column name".
  • Performanse: Kada je ime objekta u potpunosti kvalifikovano (uključujući i šemu, npr. dbo.Ucenici), SQL Server ne mora da proverava da li trenutni korisnik poseduje objekat sa tim imenom. On odmah zna gde da ga traži, što štedi resurse.

Funkcije Wildcard expansion i Qualify object names imaju snažnu sinergiju. Često se koriste zajedno kako bi se osigurao najviši kvalitet koda.

Sekcija 4: Primer 3 (Srednji nivo): Inkapsulacija Logike - Kreiranje Stored Procedure iz Običnog Koda

Kako vaši SQL upiti postaju složeniji, primetićete da se određeni delovi logike ponavljaju na više mesta u aplikaciji. Možda imate kompleksan upit za dohvatanje podataka o učenicima sa najboljim prosekom koji se koristi i na web stranici, i u mobilnoj aplikaciji, i u nedeljnom izveštaju. Kopiranje istog koda na više mesta je recept za katastrofu.

Rešenje za ovaj problem je inkapsulacija – "pakovanje" vaše SQL logike u ponovno upotrebljive objekte unutar baze podataka, kao što su uskladištene procedure (stored procedures).

Zašto "sakrivati" kod u stored procedure?

Pretvaranje običnog T-SQL koda u uskladištenu proceduru donosi niz fundamentalnih prednosti koje su stub profesionalnog razvoja baza podataka:

  • Ponovna upotrebljivost (Reusability): Ovo je primena čuvenog programerskog principa "Don't Repeat Yourself" (DRY). Umesto da se isti kod ponavlja, on se napiše jednom unutar procedure.
  • Održavanje: Kada se poslovna pravila promene (npr. "odličan uspeh" se sada računa od proseka 4.60 umesto 4.50), promenu pravite na samo jednom mestu – unutar tela procedure.
  • Bezbednost: Uskladištene procedure su vaša prva linija odbrane protiv SQL Injection napada. Umesto da aplikaciji date direktna prava da čita i piše u vaše tabele, vi joj dajete samo pravo da izvrši određenu proceduru (EXECUTE).
  • Performanse: Kada se uskladištena procedura izvrši prvi put, SQL Server kreira "plan izvršenja" (execution plan) koji se kešira i ponovo koristi, što izvršavanje čini znatno bržim.

Iako su prednosti ogromne, mnogi početnici zaziru od pisanja procedura. Tu ApexSQL Refactor stupa na scenu, automatizujući 90% posla.

Korak po korak: Od skripte do procedure

Recimo da imamo sledeći T-SQL kod koji pronalazi sve učenike čiji je prosek veći od 4.50. Da bismo ovaj kod pretvorili u proceduru:

  1. Selektujte ceo T-SQL blok u SSMS editoru.
  2. Iz menija izaberite ApexSQL -> ApexSQL Refactor -> Encapsulate code as... -> Stored Procedure.
  3. Otvoriće se dijalog prozor u kojem treba da unesete ime za novu proceduru (npr. usp_GetOdlicneUcenike).

Praktičan primer: Pametno prepoznavanje parametara

Ono što sledi nije samo prosto "pakovanje" koda. ApexSQL Refactor je dovoljno inteligentan da analizira kod koji ste selektovali.

Kod PRE (selektovan u editoru):

SELECT
UcenikID,
Ime,
Prezime,
Prosek
FROM
dbo.Ucenici
WHERE
Prosek > 4.50 -- Alat će prepoznati ovu vrednost kao kandidata za parametar
ORDER BY
Prosek DESC;

Akcija i Analiza: Kada pokrenete Encapsulate code as čarobnjak, on će prepoznati literalnu (hardkodovanu) vrednost 4.50 u WHERE klauzuli. Umesto da je ostavi takvom, on će vam ponuditi da je pretvorite u ulazni parametar. Ovo je ključni korak koji vašu logiku čini fleksibilnom i ponovno upotrebljivom.

Ova sposobnost alata da analizira kod i predloži parametre je ono što ga čini izvanrednim alatom za učenje. On vas ne uči samo da "umotate" kod, već vas uči fundamentalnom konceptu generalizacije – pisanju fleksibilnog koda koji može da reši čitavu klasu problema, a ne samo jedan specifičan slučaj.

Kod POSLE (generisan od strane alata):

CREATE PROCEDURE dbo.usp_GetOdlicneUcenike (
@MinimalniProsek DECIMAL(3, 2)
)
AS
BEGIN
SET NOCOUNT ON;

SELECT
UcenikID,
Ime,
Prezime,
Prosek
FROM
dbo.Ucenici
WHERE
Prosek > @MinimalniProsek
ORDER BY
Prosek DESC;
END

Sada, vašu logiku pozivate jednostavnom komandom:

EXEC dbo.usp_GetOdlicneUcenike @MinimalniProsek = 4.50;

Ili za drugu vrednost:

EXEC dbo.usp_GetOdlicneUcenike @MinimalniProsek = 4.80;

Dodatni savet: Inkapsulacija u View ili Funkciju

Ista tehnika se može primeniti i za kreiranje drugih objekata. Ako vaš kod samo čita podatke i ne zahteva parametre, možete ga inkapsulirati kao View. View se ponaša kao virtuelna tabela. Ako kod vrši proračun i vraća jednu vrednost, možete ga inkapsulirati kao skalarnu funkciju. ApexSQL Refactor podržava sve ove opcije unutar Encapsulate code as... menija.

Sekcija 5: Primer 4 (Napredni nivo): Refaktorisanje Strukture Baze - Bezbedno Preimenovanje Objekata

Došli smo do najnaprednije i verovatno najvažnije funkcije refaktorisanja – one koja se bavi direktnim izmenama strukture (šeme) baze podataka. Promena imena tabele ili kolone može delovati kao jednostavan zadatak, ali u realnom svetu, to je jedna od najrizičnijih operacija koje programer može da izvede.

Zašto je ručno preimenovanje "ruski rulet"?

Baza podataka nije samo kolekcija izolovanih tabela. Ona je složena mreža međusobno povezanih objekata. Ovaj koncept se naziva zavisnost (dependency). Na primer:

  • Stored procedure zavisi od tabela i kolona koje čita ili u koje upisuje.
  • View zavisi od tabela i kolona iz kojih povlači podatke.
  • Foreign Key ograničenje u jednoj tabeli zavisi od primarnog ključa u drugoj tabeli.

Kada ručno preimenujete kolonu, na primer iz Prosek u ProsecnaOcena, koristeći standardni Object Explorer u SSMS-u, vi menjate samo ime na jednom mestu. Međutim, SQL Server neće automatski pronaći i ažurirati svaki View, proceduru ili funkciju koja je koristila staro ime.

Posledica je katastrofalna. U trenutku kada aplikacija pokuša da izvrši proceduru koja i dalje traži nepostojeću kolonu Prosek, program će se srušiti sa greškom "Invalid column name".

Funkcija Safe Rename u ApexSQL Refactor-u rešava ovaj problem tako što preuzima ulogu detektiva. Pre nego što izvrši promenu, ona mapira celu mrežu zavisnosti i automatski generiše skripte za ažuriranje svakog zavisnog objekta.

Korak po korak: Hirurška preciznost uz "Safe Rename"

Zamislimo scenario: imamo tabelu Ucenici sa kolonom Prosek i proceduru usp_GetOdlicneUcenike koja koristi tu kolonu. Naš zadatak je da preimenujemo kolonu Prosek u ProsecnaOcena.

  1. U SSMS Object Explorer-u, pronađite tabelu Ucenici i proširite njenu listu kolona.
  2. Desni klik na kolonu Prosek.
  3. Iz kontekstnog menija izaberite ApexSQL Refactor -> Safe rename.
  4. U dijalog prozoru, u polje New name unesite ProsecnaOcena.
  5. Kliknite na dugme Preview.

Praktičan primer: Magija analize zavisnosti

Preview prozor koji se otvara je komandni centar ove operacije. On vam čini vidljivim ono što je inače nevidljivo – skrivene veze između objekata u vašoj bazi.

Stanje PRE:

  • Tabela Ucenici ima kolonu Prosek.
  • Procedura usp_GetOdlicneUcenike sadrži liniju: WHERE Prosek > @MinimalniProsek.

Akcija i Analiza: Nakon što ste u Preview prozoru pregledali sve zavisnosti i uverili se da je alat sve ispravno detektovao, imate dve opcije:

  • Create script: Generiše jednu veliku SQL skriptu koja sadrži i sp_rename i sve potrebne ALTER naredbe.
  • Execute: Direktno izvršava sve potrebne skripte i primenjuje promenu.

Stanje POSLE: Nakon izvršenja, dešavaju se dve stvari simultano:

  1. Kolona u tabeli Ucenici je uspešno preimenovana u ProsecnaOcena.
  2. Telo procedure usp_GetOdlicneUcenike je automatski izmenjeno i sada sadrži liniju: WHERE ProsecnaOcena > @MinimalniProsek.

Rezultat? Promena je primenjena, a ceo sistem nastavlja da radi bez ikakvih grešaka. Integritet baze podataka je sačuvan.

Dodatni savet: Šta sve može "Safe Rename"?

Ova moćna tehnika nije ograničena samo na kolone. Možete je koristiti za bezbedno preimenovanje praktično bilo kog objekta u bazi podataka:

  • Tabela
  • View-ova
  • Uskladištenih procedura
  • Funkcija
  • Parametara unutar procedura i funkcija
  • Trigera

Svaki put, proces je isti: ApexSQL Refactor će izvršiti dubinsku analizu zavisnosti i osigurati da se promena bezbedno propagira kroz ceo sistem.

Sekcija 6: Zaključak: Put ka Čistijem i Efikasnijem Kodu

Kroz četiri praktična primera, prešli smo put od osnovne higijene koda do naprednih hirurških zahvata na strukturi baze podataka. Svaki korak je predstavljao ne samo učenje nove funkcije u alatu ApexSQL Refactor, već i usvajanje fundamentalnog principa profesionalnog softverskog inženjeringa.

Rezime naučenog

Naše putovanje nas je provelo kroz četiri ključna nivoa razvoja veština:

  • Vizuelna disciplina (Formatiranje): Naučili smo da čitljivost nije luksuz, već osnova za efikasan rad i saradnju.
  • Pisanje otpornog koda (Proširivanje *): Shvatili smo da prečice poput SELECT * mogu dovesti do nestabilnosti. Eksplicitnim navođenjem kolona pišemo kod koji je robustan i otporan na buduće promene.
  • Strukturirano programiranje (Inkapsulacija): Otkrili smo moć "pakovanja" logike u uskladištene procedure, čime smo postigli ponovnu upotrebljivost, bezbednost i lakše održavanje.
  • Bezbedno upravljanje šemom (Bezbedno preimenovanje): Savladali smo najrizičniji deo posla – izmenu strukture baze. Korišćenjem alata za analizu zavisnosti, naučili smo kako da refaktorišemo šemu sa samopouzdanjem.

Refaktorisanje kao navika, ne kao zadatak

Najvažnija lekcija koju treba poneti iz ovog vodiča je da refaktorisanje nije jednokratna aktivnost. To je stalna praksa, navika koju treba integrisati u svakodnevni rad. Kao što dobar kuvar čisti za sobom dok kuva, tako i dobar programer konstantno unapređuje kod na kojem radi.

Preporučuje se da ApexSQL Refactor postane vaš stalni pratilac. Nakon što napišete blok koda, iskoristite Format SQL. Pre nego što ga sačuvate, razmislite da li bi trebalo da bude inkapsuliran. Ako primetite loše ime kolone, nemojte ga ignorisati iz straha – iskoristite Safe Rename. Vremenom, ove radnje će postati automatske, a kvalitet vašeg koda će eksponencijalno rasti.

Šta dalje?

Pokrili smo četiri ključne funkcije, ali ApexSQL Refactor nudi još mnogo toga. Kada se budete osećali sigurno sa onim što ste naučili, evo nekoliko naprednih funkcija koje vredi istražiti:

  • Split Table: Omogućava vam da veliku, nezgrapnu tabelu sa mnogo kolona podelite na dve ili više manjih, logički povezanih tabela.
  • Add Surrogate Key: Automatizuje proces zamene kompleksnog, višedelnog prirodnog ključa sa jednostavnim, jedinstvenim numeričkim (surogat) ključem.
  • Change Parameters: Omogućava da bezbedno dodajete, uklanjate, preimenujete ili menjate redosled parametara u procedurama i funkcijama, automatski ažurirajući sav kod koji ih poziva.

Ulaganje vremena u učenje i primenu principa čistog koda i refaktorisanja je jedna od najboljih investicija koju možete napraviti za svoju buduću karijeru. Alati poput ApexSQL Refactor-a su tu da taj proces učine lakšim, bržim i bezbednijim. Nastavite da učite, eksperimentišete i, pre svega, pišite kod na koji ćete biti ponosni.

Dobrodošli u napredni kurs veb razvoja, osmišljen da vaše postojeće znanje o HTML-u, CSS-u i JavaScriptu podigne na profesionalni nivo. Iako već poznajete osnove, svet modernog veb razvoja zahteva dublje razumevanje ne samo kako nešto napraviti, već i zašto se to radi na određeni način. Ovaj kurs je strukturiran kao serija postepenih predavanja koja će vas voditi od temelja do izrade složenih, interaktivnih veb aplikacija.

Pregled Tehnologija

Pre nego što zaronimo u napredne koncepte, važno je da se podsetimo fundamentalnih uloga svake od tri ključne tehnologije koje čine moderni veb:

  • HTML (HyperText Markup Language): Predstavlja kostur i strukturu veb stranice. Njegova primarna uloga nije da definiše kako će nešto izgledati, već šta taj sadržaj znači. Kroz semantički HTML, mi dajemo smisao sadržaju, što je ključno za pristupačnost i optimizaciju za pretraživače.
  • CSS (Cascading Style Sheets): Zadužen je za vizuelnu prezentaciju – izgled, boje, fontove i, što je najvažnije, raspored elemenata na stranici. Dok HTML definiše šta je sadržaj, CSS određuje kako će taj sadržaj biti prikazan korisniku.
  • JavaScript (JS): Daje život veb stranici. To je programski jezik koji upravlja ponašanjem i interaktivnošću. Kada korisnik klikne na dugme, popuni formu ili skroluje kroz galeriju, JavaScript je taj koji obrađuje te akcije i dinamički menja sadržaj i stil stranice.

Struktura Predavanja

Ovaj vodič je podeljen na šest ključnih delova, sa ciljem da postepeno gradimo složenost i integrišemo znanja:

  1. Napredni HTML: Fokusiraćemo se na semantiku, pravilnu strukturu dokumenata, kreiranje kompleksnih tabela i formi, kao i na ugrađivanje multimedije.
  2. Osnove CSS-a: Proći ćemo kroz fundamentalne koncepte stilizovanja, uključujući osnovne selektore, rad sa bojama, i ključni "Box Model".
  3. Napredni CSS: Ovde ćemo ovladati modernim tehnikama za raspored elemenata, Flexbox i Grid, kao i kreiranjem responzivnog dizajna i upotrebom naprednih selektora.
  4. JavaScript - Interaktivnost i Podaci: Naučićemo kako da manipulišemo DOM-om, upravljamo događajima, radimo sa podacima i komuniciramo sa serverima putem asinhronih poziva.
  5. Sinergija HTML-a i CSS-a: Kroz praktične primere, videćemo kako se kombinovanjem znanja iz prva dva dela grade kompletne, vizuelno privlačne statičke stranice.
  6. Puna Snaga Veba (HTML, CSS i JS): U finalnom delu, spojićemo sve tri tehnologije da bismo kreirali potpuno funkcionalne, dinamičke veb aplikacije.

Svaka lekcija sadrži teorijsku osnovu i detaljno objašnjen praktičan primer sa kompletnim kodom. Krenimo na putovanje od osnova do majstorstva u veb razvoju.

Deo 1: Napredni HTML - Struktura i Semantika

Dobar HTML nije samo onaj koji ispravno prikazuje sadržaj, već onaj koji je smislen, pristupačan svima i optimizovan za pretraživače. U ovom delu, prelazimo sa prostog korišćenja tagova na njihovu pravilnu upotrebu, postavljajući temelje za profesionalni veb razvoj.

Lekcija 1.1: Semantički HTML za Smislenu Strukturu

Teorija

Semantika u HTML-u se odnosi na korišćenje tagova koji precizno opisuju značenje i ulogu sadržaja koji obuhvataju, umesto da samo diktiraju njegov izgled. Na primer, <h1> tag ne govori browseru samo "učini ovaj tekst velikim", već "ovo je naslov najvišeg nivoa na stranici". Korišćenje semantičkih elemenata donosi tri ključna benefita:

  • Pristupačnost (Accessibility): Asistivne tehnologije, poput čitača ekrana koje koriste slepe i slabovide osobe, oslanjaju se na semantičku strukturu da bi razumele i interpretirale sadržaj. Takozvani "landmark" elementi kao što su <header>, <nav>, <main> i <footer> deluju kao putokazi, omogućavajući korisnicima da brzo preskaču na željene delove stranice, umesto da moraju da slušaju ceo sadržaj redom.
  • Optimizacija za pretraživače (SEO): Pretraživači poput Google-a koriste složene algoritme da bi razumeli kontekst i relevantnost sadržaja na stranici. Kada je sadržaj pravilno strukturiran pomoću semantičkih tagova, pretraživači ga lakše indeksiraju i preciznije rangiraju u rezultatima pretrage.
  • Održivost koda: Kod koji koristi semantičke elemente je intuitivniji i lakši za čitanje, kako za vas tako i za druge programere. Jasno je koja je uloga svakog dela stranice, što olakšava buduće izmene i održavanje.

Glavni strukturni elementi koje je uveo HTML5 su:

  • <header>: Predstavlja zaglavlje stranice ili sekcije, obično sadrži logo, naslov i navigaciju.
  • <nav>: Namenjen je za glavne navigacione linkove.
  • <main>: Obuhvata glavni, jedinstveni sadržaj stranice. Trebalo bi da postoji samo jedan <main> element po stranici.
  • <article>: Definiše samostalan, nezavisan deo sadržaja koji bi imao smisla i van konteksta stranice, kao što je blog post, novinski članak ili komentar na forumu.
  • <section>: Grupiše tematski povezan sadržaj. Za razliku od <article>, sadržaj unutar <section> ne mora nužno biti samostalan.
  • <aside>: Koristi se za sadržaj koji je tangencijalno povezan sa glavnim sadržajem, poput bočne trake (sidebar) sa dodatnim informacijama ili linkovima.
  • <footer>: Predstavlja podnožje stranice ili sekcije, obično sadrži informacije o autoru, autorskim pravima i kontakt linkove.

Izbor između <div class="button"> i <button> nije samo estetski, već fundamentalno funkcionalan. Dok <div> može biti stilizovan da izgleda kao dugme, on ne nosi nikakvo semantičko značenje. Čitač ekrana će ga pročitati kao običan tekst, a korisnik neće znati da je to interaktivni element. S druge strane, <button> element sa sobom nosi ugrađenu funkcionalnost: može se fokusirati tastaturom, aktivirati tasterom "Enter" i čitači ekrana ga ispravno najavljuju kao "dugme". Korišćenje ne-semantičkog HTML-a stvara "tehnički dug" pristupačnosti, efektivno isključujući deo korisnika.

Primer 1: Struktura Blog Posta

Cilj: Kreirati kostur za tipičan blog post koristeći isključivo semantičke elemente kako bi se demonstrirala pravilna struktura.

<!DOCTYPE html>
<html lang="sr">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Moj Blog Post</title>
</head>
<body>

<header>
<h1>Moj Blog</h1>
<nav>
<ul>
<li><a href="#">Početna</a></li>
<li><a href="#">O meni</a></li>
<li><a href="#">Kontakt</a></li>
</ul>
</nav>
</header>

<main>
<article>
<header>
<h2>Naslov Blog Posta</h2>
<p>Objavljeno: 15. oktobar 2023. od strane Jovana Petrovića</p>
</header>

<section>
<h3>Uvod</h3>
<p>Ovo je uvodni pasus našeg blog posta koji objašnjava o čemu će se govoriti...</p>
</section>

<section>
<h3>Glavna tema</h3>
<p>Ovde se nalazi glavni deo teksta, sa detaljnim objašnjenjima...</p>

<figure>
<img src="slika.jpg" alt="Opis slike za pristupačnost">
<figcaption>Slika 1: Detaljan opis onoga što je na slici.</figcaption>
</figure>

<p>Nastavak teksta nakon slike...</p>
</section>
</article>

<aside>
<h3>Povezani članci</h3>
<ul>
<li><a href="#">Stariji post 1</a></li>
<li><a href="#">Stariji post 2</a></li>
</ul>
</aside>
</main>

Analiza Koda:

  • <header> na vrhu sadrži naslov sajta i <nav> za globalnu navigaciju.
  • <main> obuhvata sav jedinstveni sadržaj ove stranice.
  • <article> je idealan za blog post jer je to samostalna celina. Unutar njega, imamo sopstveni <header> sa naslovom posta.
  • <section> elementi se koriste za tematsko grupisanje delova unutar članka (Uvod, Glavna tema).
  • <figure> i <figcaption> se koriste za semantičko povezivanje slike sa njenim opisom.
  • <aside> sadrži povezane linkove, što je tipična upotreba za ovaj element.
  • <footer> na dnu stranice sadrži informacije o autorskim pravima.

Lekcija 1.2: Kreiranje Kompleksnih Tabela za Podatke

Teorija

HTML tabele su namenjene isključivo za prikazivanje tabelarnih podataka. Nikada ih ne treba koristiti za layout stranice. Za napredne tabele, koristimo: <thead>, <tbody>, <tfoot> za grupisanje redova; colspan i rowspan za spajanje ćelija; <caption> za naslov tabele; i scope atribut za pristupačnost.

Bez atributa scope, čitač ekrana bi korisniku pružio nepovezane podatke. Na primer, pročitao bi samo "30 evra". Sa scope="col" na zaglavlju "Cena", čitač ekrana može jasno reći: "Kolona: Cena, Vrednost: 30 evra", pretvarajući niz brojeva u smislen izveštaj.

Primer 2: Tabela za Praćenje Troškova

Cilj: Napraviti tabelu koja prikazuje mesečne troškove, sa sumarnim redom u futeru i pravilno povezanim zaglavljima i podacima radi pristupačnosti.

<!DOCTYPE html>
<html lang="sr">
<head>...</head>
<body>

<h1>Moji mesečni troškovi</h1>

<table>
<caption>Pregled troškova za Oktobar 2023.</caption>

<thead>
<tr>
<th scope="col">Stavka</th>
<th scope="col">Datum</th>
<th scope="col">Kategorija</th>
<th scope="col">Iznos (€)</th>
</tr>
</thead>

<tbody>
<tr>
<th scope="row">Račun za struju</th>
<td>05.10.2023.</td>
<td>Komunalije</td>
<td>50.00</td>
</tr>
<tr>
<th scope="row">Nabavka namirnica</th>
<td>10.10.2023.</td>
<td>Hrana</td>
<td>120.50</td>
</tr>
</tbody>

<tfoot>
<tr>
<td colspan="3">Ukupno:</td>
<td>200.50</td>
</tr>
</tfoot>
</table>

</body>
</html>

Analiza Koda:

  • <caption> jasno opisuje svrhu tabele.
  • <thead> sadrži zaglavlja kolona (<th>) sa scope="col".
  • <tbody> sadrži redove sa podacima. Prva ćelija je zaglavlje reda sa scope="row".
  • <tfoot> sadrži sumarni red, gde prva ćelija koristi colspan="3" da se prostire preko tri kolone.

Lekcija 1.3: Izgradnja Naprednih Formi za Unos Podataka

Teorija

Forme su ključni element za interakciju. Pravilna struktura forme poboljšava korisničko iskustvo i osigurava pristupačnost. Koristimo <fieldset> i <legend> za grupisanje polja, i uvek povezujemo <label> sa svojim poljem (eksplicitno sa for i id, ili implicitno ugnežđavanjem).

HTML5 tipovi inputa (email, tel, number, date) donose ugrađenu validaciju i optimizovanu tastaturu na mobilnim uređajima.

Primer 3: Forma za Registraciju

Cilj: Kreirati detaljnu formu za registraciju sa različitim tipovima polja, grupisanim u logičke celine radi bolje preglednosti.

<!DOCTYPE html>
...
<body>
<form>
<h1>Forma za registraciju</h1>
<p>Polja označena sa * su obavezna.</p>

<fieldset>
<legend>Lični podaci</legend>
<p>
<label for="ime">Ime *:</label>
<input type="text" id="ime" name="korisnicko_ime" required>
</p>
<p>
<label for="email">Email *:</label>
<input type="email" id="email" name="korisnicki_email" required>
</p>
</fieldset>

<fieldset>
<legend>Dodatne informacije</legend>
<p>
<label for="godine">Godine:</label>
<input type="number" id="godine" name="godine" min="18" max="100">
</p>
</fieldset>

<button type="submit">Registruj se</button>
</form>
</body>
</html>

Analiza Koda:

  • Forma je podeljena u <fieldset> elemente, svaki sa svojim <legend> naslovom.
  • Korišćeni su različiti input tipovi: text, email, number. Atribut required označava obavezna polja.
  • <select> se koristi za padajuću listu, a <textarea> za unos dužeg teksta.

Lekcija 1.4: Ugrađivanje Multimedije: <video> i <audio>

Teorija

HTML5 je standardizovao ugrađivanje audio i video sadržaja. Koristimo atribute controls, autoplay, loop, i poster. Za maksimalnu kompatibilnost, koristimo <source> element da ponudimo više formata (npr. MP4, WebM). Element <track> omogućava dodavanje tekstualnih traka (titlova), što je ključno za pristupačnost.

Primer 4: Video Plejer sa Titlovima

Cilj: Ugraditi video plejer koji ima kontrole, nudi više formata radi kompatibilnosti i pruža opciju za engleske titlove.

<!DOCTYPE html>
...
<body>

<h1>Primer sa Video Plejerom</h1>

<video controls poster="poster-slika.jpg" width="620">
<source src="moj-video.webm" type="video/webm">
<source src="moj-video.mp4" type="video/mp4">

<track
src="titlovi_en.vtt"
kind="subtitles"
srclang="en"
label="English">

<p>Vaš browser ne podržava HTML video.</p>
</video>

</body>
</html>

Analiza Koda:

  • <video> element ima controls atribut za prikazivanje kontrola i poster za sliku pre pokretanja.
  • Unutar <video> taga, navedena su dva <source> elementa za različite formate.
  • <track> element definiše putanju do .vtt fajla sa titlovima.
  • Paragraf na kraju služi kao "fallback" sadržaj i prikazuje se samo ako browser uopšte ne prepoznaje <video> tag.

Deo 2: Osnove CSS-a - Uvod u Stilizovanje

Pre nego što zaronimo u napredne tehnike rasporeda kao što su Flexbox i Grid, ključno je da utvrdimo temelje CSS-a. U ovom delu, proći ćemo kroz osnovne koncepte koji vam omogućavaju da "oživite" HTML strukturu dodavanjem boja, stilova i osnovnog rasporeda.

Lekcija 2.1: Osnovni Selektori i Svojstva

Teorija

CSS funkcioniše tako što se primenjuju pravila na HTML elemente. Svako pravilo se sastoji od selektora (koji element ciljamo) i bloka deklaracija (kako želimo da ga stilizujemo). Svaka deklaracija se sastoji od para svojstvo: vrednost;.

  • Selektor taga (Type Selector): Cilja sve elemente određenog HTML taga. Na primer, p će selektovati sve paragrafe na stranici.
  • Selektor klase (Class Selector): Cilja sve elemente koji imaju određeni class atribut. U CSS-u se piše sa tačkom ispred imena klase (npr. .istaknuto). Jedna klasa se može primeniti na više elemenata.
  • ID selektor (ID Selector): Cilja jedan, jedinstveni element koji ima određeni id atribut. U CSS-u se piše sa tarabom ispred imena ID-a (npr. #glavni-naslov). Svaki ID mora biti jedinstven na stranici.

Primer 1: Stilizovanje Teksta

Cilj: Primeniti različite stilove na naslov, jedan istaknuti paragraf i jedan poseban paragraf koristeći sva tri osnovna tipa selektora.

Dobrodošli na moju stranicu

Ovo je običan paragraf teksta.

Ovaj paragraf je istaknut i važan.

HTML kod:

<h1 id="glavni-naslov">Dobrodošli na moju stranicu</h1>
<p>Ovo je običan paragraf teksta.</p>
<p class="istaknuto">Ovaj paragraf je istaknut i važan.</p>

CSS kod:

/* Selektor taga */
p {
font-family: Arial, sans-serif;
font-size: 16px;
}

/* ID selektor */
#glavni-naslov {
color: #0066cc;
text-align: center;
}

/* Selektor klase */
.istaknuto {
color: crimson;
font-style: italic;
}

Lekcija 2.2: Rad sa Bojama i Pozadinama

Teorija

Dva najosnovnija svojstva za stilizovanje su color (boja teksta) i background-color (boja pozadine). Boje se mogu definisati imenima (red), heksadecimalnim kodovima (#FF0000) ili RGB vrednostima (rgb(255, 0, 0)).

Lekcija 2.3: Uvod u CSS Box Model

Teorija

Svaki HTML element je pravougaona kutija. CSS Box Model opisuje njene delove, idući od unutra ka spolja: Sadržaj (Content), Unutrašnja margina (Padding), Okvir (Border) i Spoljašnja margina (Margin).

Ovo je sadržaj unutar kutije.

padding: 20px; /* Unutrašnja margina */

Deo 3: Napredni CSS - Layout i Stilovi

CSS je evoluirao od jednostavnog alata za stilizovanje u moćan sistem za kreiranje kompleksnih i responzivnih rasporeda. U ovom delu, fokusiraćemo se na dva moderna stuba CSS layout-a – Flexbox i Grid – koji su fundamentalni za izradu današnjih veb stranica.

Lekcija 3.1: Ovladavanje Flexbox Layout-om

Teorija

Flexbox (Flexible Box Layout) je model dizajniran za jednodimenzionalni raspored elemenata – bilo u redu (horizontalno) ili u koloni (vertikalno). Idealan je za raspoređivanje elemenata unutar jedne komponente, kao što su stavke u navigaciji ili polja u formi.

Osnovna terminologija:
  • Flex kontejner: Element na koji se primeni display: flex;.
  • Flex item-i: Direktna deca flex kontejnera.
  • Glavna osa (main axis): Smer u kojem se raspoređuju item-i (podrazumevano horizontalno).
  • Poprečna osa (cross axis): Osa koja je upravna na glavnu osu (podrazumevano vertikalno).
Ključna svojstva kontejnera:
  • flex-direction: Određuje smer glavne ose (row, column, itd.).
  • justify-content: Kontroliše poravnanje item-a duž glavne ose (center, space-between, itd.).
  • align-items: Kontroliše poravnanje item-a duž poprečne ose.
  • flex-wrap: Dozvoljava item-ima da pređu u novi red.

Primer 1: Fleksibilna Navigaciona Traka

Cilj: Kreirati standardnu navigacionu traku gde je logo poravnat levo, a navigacioni linkovi desno, pri čemu su svi elementi vertikalno centrirani.

MojLogo
  • Početna
  • Usluge
  • Kontakt

HTML kod:

<nav class="navigation-bar">
<div class="logo">
<a href="#">MojLogo</a>
</div>
<ul class="nav-links">
<li><a href="#">Početna</a></li>
<li><a href="#">Usluge</a></li>
<li><a href="#">Kontakt</a></li>
</ul>
</nav>

CSS kod:

.navigation-bar {
display: flex;
justify-content: space-between; /* Postavlja logo levo, a linkove desno */
align-items: center; /* Vertikalno centrira sve elemente */
background-color: #333;
padding: 10px 20px;
color: white;
}

.nav-links {
display: flex; /* Pretvara i listu u flex kontejner */
list-style: none;
margin: 0;
padding: 0;
}

.nav-links li {
margin-left: 20px; /* Dodaje razmak između linkova */
}

a {
color: white;
text-decoration: none;
}

Analiza Koda:

  • display: flex; na .navigation-bar kontejneru postavlja div.logo i ul.nav-links kao flex item-e.
  • justify-content: space-between; uzima sav višak horizontalnog prostora i gura ga između item-a, efektivno postavljajući prvi item (logo) na početak, a drugi (lista linkova) na kraj.
  • align-items: center; osigurava da su i logo i linkovi savršeno poravnati po vertikalnoj sredini, bez obzira na njihovu visinu.
  • Dodatni display: flex; na ul.nav-links je potreban da bi se <li> elementi poređali horizontalno jedan pored drugog.

Lekcija 3.2: Moć CSS Grid-a

Teorija

CSS Grid Layout je revolucionarni dvodimenzionalni sistem za raspored, koji omogućava istovremenu kontrolu nad redovima i kolonama. Savršen je za kreiranje kompleksnih layout-a cele stranice.

Osnovna terminologija:
  • Grid kontejner: Element na koji se primeni display: grid;.
  • Grid linije: Horizontalne i vertikalne linije koje formiraju mrežu.
  • Grid trake (tracks): Prostor između dve linije (red ili kolona).
  • Grid ćelija: Najmanja jedinica mreže.
  • Grid oblast (area): Pravougaoni prostor koji se prostire preko jedne ili više ćelija.
Definisanje mreže:
  • grid-template-columns i grid-template-rows: Definišu broj i veličinu kolona i redova.
  • fr jedinica: Predstavlja frakciju (deo) dostupnog prostora, omogućavajući fleksibilne i proporcionalne rasporede.
Pozicioniranje elemenata:
  • Pomoću linija: Korišćenjem grid-column-start, grid-column-end, itd.
  • Pomoću imenovanih oblasti: Definisanjem grid-template-areas na kontejneru i dodeljivanjem grid-area svojstva svakom item-u. Ovaj pristup je izuzetno intuitivan i vizuelan.

Grid i Flexbox nisu rivali, već partneri. Grid je idealan za makro-layout (raspored glavnih delova stranice kao što su header, main, sidebar, footer), dok je Flexbox savršen za mikro-layout (raspoređivanje elemenata unutar tih delova). Na primer, možete koristiti Grid da pozicionirate header, a zatim unutar tog header-a koristiti Flexbox da poravnate logo i navigaciju.

Primer 2: "Holy Grail" Layout Stranice

Cilj: Kreirati klasični "Holy Grail" layout (header, footer, centralni sadržaj i dva bočna panela) koristeći CSS Grid i imenovane oblasti.

Header
Navigation
Main Content
Sidebar
Footer

HTML kod:

<div class="container">
<header class="header">Header</header>
<nav class="nav">Navigation</nav>
<main class="main">Main Content</main>
<aside class="sidebar">Sidebar</aside>
<footer>Footer</footer>
</div>

CSS kod:

.container {
display: grid;
height: 100vh; /* Da se layout prostire preko celog ekrana */
grid-template-columns: 200px 1fr 200px; /* Leva i desna fiksne, srednja fleksibilna */
grid-template-rows: auto 1fr auto; /* Header/footer visine sadržaja, main fleksibilan */
grid-template-areas:
"header header header"
"nav main sidebar"
"footer footer footer";
gap: 10px; /* Razmak između elemenata */
}

.header { grid-area: header; }
.nav { grid-area: nav; }
.main { grid-area: main; }
.sidebar { grid-area: sidebar; }
footer { grid-area: footer; }

Analiza Koda:

  • display: grid; na .container elementu ga pretvara u grid kontejner.
  • grid-template-columns definiše tri kolone: prva i treća su fiksne širine 200px, dok druga (1fr) zauzima sav preostali raspoloživi prostor.
  • grid-template-rows definiše tri reda: prvi (auto) i treći (auto) će biti visoki koliko i njihov sadržaj, dok drugi (1fr) zauzima sav preostali vertikalni prostor. Ovo osigurava da se footer "lepi" za dno stranice ako sadržaj nije dovoljno visok.
  • grid-template-areas je ključni deo. On vizuelno, ASCII-art stilom, definiše raspored. Ime "header" se ponavlja tri puta, što znači da će se element sa grid-area: header; prostirati preko sve tri kolone u prvom redu. Isto važi i za "footer".
  • Na kraju, svakom HTML elementu se dodeljuje odgovarajuće ime oblasti pomoću grid-area svojstva.

Lekcija 3.3: Responzivni Dizajn sa Media Queries

Teorija

Responzivni veb dizajn je praksa kreiranja veb stranica koje pružaju optimalno iskustvo gledanja i interakcije na širokom spektru uređaja (od mobilnih telefona do desktop računara).

  • Media Queries: Osnovni alat za postizanje responzivnosti. To je CSS tehnika koja omogućava primenu stilova samo kada su određeni uslovi ispunjeni. Sintaksa @media se koristi za definisanje bloka stilova koji će se primeniti pod određenim uslovima.
  • Breakpoints (prelomne tačke): Specifične širine viewport-a na kojima se layout stranice menja. Najčešće se koriste uslovi min-width (za stilove koji se primenjuju iznad određene širine) i max-width (za stilove koji se primenjuju ispod određene širine).
  • Mobile-First pristup: Moderna i preporučena praksa gde se prvo pišu osnovni CSS stilovi za najmanje ekrane (mobilne telefone). Zatim se koriste min-width media queries da se dodaju ili modifikuju stilovi za veće ekrane (tablete, desktop). Ovaj pristup osigurava da mobilni uređaji učitavaju samo neophodan CSS, što poboljšava performanse.

  • Unutar media query-ja, flex-direction se menja u row, što postavlja sliku i informacije jedno pored drugog.
  • Svojstva flex: 1 i flex: 2 se koriste da bi se definisao proporcionalni odnos širine između slike i teksta na većim ekranima.

Lekcija 3.4: Napredni Selektori i Pseudo-elementi

Teorija

Pored osnovnih selektora (po tagu, klasi, ID-u), CSS nudi napredne selektore koji omogućavaju preciznije ciljanje elemenata bez potrebe za dodavanjem dodatnih klasa ili ID-eva.

  • Atributni selektori: Omogućavaju selekciju elemenata na osnovu njihovih atributa i vrednosti. Primer: [target="_blank"] selektuje linkove koji se otvaraju u novom tabu, dok [href$=".pdf"] selektuje linkove koji vode ka PDF fajlovima.
  • Pseudo-klase: Selektuju elemente na osnovu njihovog stanja ili pozicije u DOM-u.
    • Stanja: :hover, :focus, :active.
    • Strukturne: :first-child, :last-child, :nth-child(n) (selektuje svaki n-ti element).
  • Pseudo-elementi: Omogućavaju stilizovanje delova elementa koji ne postoje kao zasebni elementi u DOM-u.
    • ::before i ::after: Kreiraju "virtuelne" elemente pre ili posle sadržaja. Zahtevaju content svojstvo.
    • ::selection: Stilizuje deo teksta koji je korisnik selektovao mišem.

Primer 4: Interaktivna FAQ Sekcija (bez JavaScripta)

Cilj: Kreirati FAQ (često postavljana pitanja) sekciju gde se klikom na pitanje otvara odgovor, koristeći samo HTML i CSS, bez ijedne linije JavaScripta.

Flexbox je jednodimenzionalni sistem za raspored (red ili kolona), dok je Grid dvodimenzionalni (redovi i kolone istovremeno).

To je upotreba HTML tagova koji opisuju značenje sadržaja, a ne samo njegov izgled.

HTML kod:

<div class="faq-item">
<input type="checkbox" id="faq1" class="faq-toggle" hidden>
<label for="faq1" class="faq-question">Koja je razlika...?</label>
<div class="faq-answer">
<p>Flexbox je jednodimenzionalni sistem...</p>
</div>
</div>

CSS kod:

.faq-toggle { display: none; }

.faq-question::after {
content: '+';
transition: transform 0.3s;
}

.faq-answer {
max-height: 0;
overflow: hidden;
transition: max-height 0.3s ease-out;
}

/* Kada je checkbox čekiran... */
.faq-toggle:checked + .faq-question::after {
/*...rotiramo '+' u 'x' */
transform: rotate(45deg);
}

.faq-toggle:checked + .faq-question + .faq-answer {
/*...prikazujemo odgovor animacijom */
max-height: 1000px;
}

Analiza Koda:

  • "Checkbox Hack": Trik se zasniva na skrivenom <input type="checkbox"> i <label> elementu koji je povezan sa njim. Klikom na labelu (pitanje), čekira se i skriva checkbox.
  • Adjacent Sibling Combinator (+): Ovo je srž mehanizma. Selektor .faq-toggle:checked + .faq-question cilja pitanje koje se nalazi odmah nakon čekiranog checkbox-a.
  • Pseudo-element ::after: Koristi se za prikaz + simbola. Kada je checkbox čekiran, transform: rotate(45deg); ga pretvara u x simbol, pružajući vizuelni feedback.
  • Animacija odgovora: Odgovor (.faq-answer) je podrazumevano skriven sa max-height: 0 i overflow: hidden. Kada je checkbox čekiran, selektor .faq-toggle:checked + .faq-question + .faq-answer cilja odgovor i postavlja mu max-height na veliku vrednost. Svojstvo transition omogućava glatku animaciju otvaranja i zatvaranja.

Deo 4: JavaScript - Interaktivnost i Podaci

JavaScript je jezik koji udahnjuje život veb stranicama. U ovom delu, naučićemo kako da dinamički menjamo sadržaj stranice, reagujemo na akcije korisnika, radimo sa strukturama podataka i komuniciramo sa eksternim servisima.

Lekcija 4.1: Dinamička Manipulacija DOM-om

Teorija

DOM (Document Object Model) je programski interfejs za HTML dokumente. On predstavlja stranicu kao stablo objekata, gde svaki HTML element, atribut i tekstualni sadržaj postaje "čvor" (node) u tom stablu. JavaScript može da pristupi ovom stablu i da ga menja, što nam omogućava da dinamički ažuriramo stranicu bez potrebe za ponovnim učitavanjem.

Selekcija elemenata:
  • document.querySelector('selector'): Vraća prvi element koji odgovara datom CSS selektoru.
  • document.querySelectorAll('selector'): Vraća NodeList (kolekciju sličnu nizu) svih elemenata koji odgovaraju selektoru.
Kreiranje i dodavanje elemenata:
  • document.createElement('tagName'): Kreira novi HTML element.
  • parentElement.appendChild(childElement): Dodaje kreirani element kao poslednje dete roditeljskog elementa.
Modifikacija elemenata:
  • element.textContent: Menja tekstualni sadržaj elementa (bezbedan način).
  • element.innerHTML: Menja kompletan HTML sadržaj unutar elementa. Oprez: Koristiti sa pažnjom jer može dovesti do sigurnosnih propusta (XSS).
  • element.classList.add/remove/toggle('className'): Bezbedan i preporučen način za manipulaciju klasama.

Primer 1: "Shopping List" Aplikacija

Cilj: Napraviti jednostavnu aplikaciju gde korisnik može da unese stavku u polje, klikne na dugme i doda je na listu. Svaka stavka na listi takođe ima dugme za brisanje.

Moja lista za kupovinu

    HTML kod:

    <h1>Moja lista za kupovinu</h1>
    <div>
    <label for="item">Unesite novu stavku:</label>
    <input type="text" id="item">
    <button>Dodaj stavku</button>
    </div>
    <ul id="shopping-list"></ul>

    JavaScript kod:

    // 1. Selektovanje potrebnih DOM elemenata
    const input = document.querySelector('#item');
    const addButton = document.querySelector('button');
    const list = document.querySelector('#shopping-list');

    // 2. Dodavanje event listener-a na dugme
    addButton.addEventListener('click', () => {
    const itemText = input.value; // Uzimamo vrednost iz input polja
    if (itemText.trim() === '') { // Proveravamo da li je polje prazno
    alert('Molimo unesite naziv stavke.');
    return;
    }
    input.value = ''; // Praznimo input polje

    // 3. Kreiranje novih elemenata
    const listItem = document.createElement('li');
    const itemSpan = document.createElement('span');
    const deleteButton = document.createElement('button');

    // 4. Postavljanje sadržaja i atributa
    itemSpan.textContent = itemText;
    deleteButton.textContent = 'Obriši';

    // 5. Povezivanje elemenata
    listItem.appendChild(itemSpan);
    listItem.appendChild(deleteButton);
    list.appendChild(listItem);

    // 6. Dodavanje funkcionalnosti za brisanje
    deleteButton.addEventListener('click', () => {
    list.removeChild(listItem);
    });

    // 7. Fokusiranje input polja za sledeći unos
    input.focus();
    });

    Analiza Koda:

    Ovaj primer demonstrira ceo ciklus DOM manipulacije:

    • Selekcija: Prvo selektujemo sve elemente sa kojima ćemo raditi.
    • Događaj: Koristimo addEventListener da bismo pokrenuli logiku kada korisnik klikne na dugme.
    • Kreiranje: document.createElement() se koristi za kreiranje <li>, <span> i <button> elemenata u memoriji.
    • Modifikacija: textContent se koristi za postavljanje teksta novih elemenata.
    • Dodavanje: appendChild() se koristi da se novi elementi dodaju u DOM stablo.
    • Uklanjanje: removeChild() se poziva na roditeljskom elementu (list) da bi se uklonio listItem.

    Lekcija 4.2: Upravljanje Događajima (Event Handling)

    Teorija

    Događaji (events) su signali koje browser šalje kada se nešto značajno desi na stranici – bilo da je to akcija korisnika (klik mišem, pritisak tastera) ili događaj samog browsera (stranica se učitala, prozor je promenio veličinu).

    • Slušači događaja (Event Listeners): Moderni i preporučeni način za reagovanje na događaje je korišćenje addEventListener() metode. Ona prihvata dva glavna argumenta: naziv događaja (npr. 'click', 'mouseover') i callback funkciju koja će se izvršiti kada se događaj desi.
    • Event objekat: Kada se event handler funkcija izvrši, ona automatski dobija event objekat kao argument. Ovaj objekat sadrži korisne informacije, kao što je event.target (element na kojem se događaj desio) i omogućava kontrolu ponašanja, npr. sa event.preventDefault().

    Lekcija 4.3: Rad sa Podacima: Nizovi i Objekti

    Teorija

    U modernom front-end razvoju, retko radimo sa statičkim HTML-om. Umesto toga, dinamički generišemo sadržaj na osnovu podataka. Dve osnovne strukture za podatke u JavaScriptu su objekti i nizovi.

    • Objekti: Predstavljaju kolekciju ključ-vrednost parova. Idealni su za opisivanje jedne celine sa više karakteristika, npr. jednog korisnika { ime: 'Pera', prezime: 'Peric', godine: 30 }.
    • Nizovi: Predstavljaju uređenu listu vrednosti. Mogu sadržati bilo koji tip podataka.
    • Nizovi objekata: Ovo je najčešća struktura podataka sa kojom ćete raditi. Na primer, lista korisnika, proizvoda, ili postova na blogu.
    Moderne metode nizova:
    • forEach(item => { ... }): Izvršava funkciju za svaki element niza.
    • map(item => { ... }): Kreira novi niz transformišući svaki element originalnog niza.
    • filter(item => { ... }): Kreira novi niz koji sadrži samo elemente koji zadovoljavaju određeni uslov.

    Primer 3: Dinamičko Generisanje i Filtriranje Liste Korisnika

    Cilj: Imati predefinisan niz objekata (korisnika) i dinamički ih prikazati na stranici. Dodati polje za pretragu koje će u realnom vremenu filtrirati prikazane korisnike.

      HTML kod:

      <input type="text" id="search-box" placeholder="Pretraži korisnike...">
      <ul id="user-list"></ul>

      JavaScript kod:

      const users = [
      { id: 1, name: 'Ana Anić', email: 'ana.anic@example.com' },
      { id: 2, name: 'Marko Marković', email: 'marko.markovic@example.com' },
      { id: 3, name: 'Petar Petrović', email: 'petar.petrovic@example.com' },
      { id: 4, name: 'Jovana Jovanović', email: 'jovana.jovanovic@example.com' }
      ];

      const userList = document.querySelector('#user-list');
      const searchBox = document.querySelector('#search-box');

      function renderUsers(userArray) {
      userList.innerHTML = ''; // Prvo praznimo listu
      userArray.forEach(user => {
      const li = document.createElement('li');
      li.textContent = `${user.name} (${user.email})`;
      userList.appendChild(li);
      });
      }

      // Event listener za pretragu
      searchBox.addEventListener('input', (event) => {
      const searchTerm = event.target.value.toLowerCase();

      const filteredUsers = users.filter(user => {
      return user.name.toLowerCase().includes(searchTerm) ||
      user.email.toLowerCase().includes(searchTerm);
      });

      renderUsers(filteredUsers);
      });

      // Inicijalno renderovanje svih korisnika
      renderUsers(users);

      Analiza Koda:

      • Niz users sadrži podatke.
      • Funkcija renderUsers je odgovorna za prikazivanje podataka u DOM-u. Ona prima niz kao argument, što je čini ponovno upotrebljivom.
      • Na searchBox je dodat input event listener, koji se aktivira pri svakoj promeni vrednosti u polju.
      • Unutar listener-a, filter() metoda se koristi za kreiranje novog niza filteredUsers koji sadrži samo korisnike čije ime ili email odgovaraju terminu za pretragu.
      • Nakon filtriranja, renderUsers se ponovo poziva, ali ovog puta sa filtriranim nizom, što dinamički ažurira prikaz na stranici.

      Lekcija 4.4: Uvod u Asinhroni JavaScript: Fetch API

      Teorija

      Većina operacija u JavaScriptu je sinhrona – izvršavaju se jedna za drugom. Međutim, operacije poput mrežnih zahteva (dohvatanje podataka sa servera) mogu trajati dugo. Asinhroni model rešava ovaj problem tako što omogućava da se dugotrajne operacije pokrenu u pozadini, bez "zamrzavanja" korisničkog interfejsa.

      • Promises (Obećanja): Promise je objekat koji predstavlja konačni ishod (uspeh ili neuspeh) asinhrone operacije.
      • Fetch API: Moderni, ugrađeni browser API za slanje HTTP zahteva. Baziran je na Promise-ima. Poziv fetch(url) vraća Promise koji se razrešava u Response objekat.
      • async/await sintaksa: Moderniji i čitljiviji način za rad sa Promise-ima. Ključna reč async se stavlja ispred definicije funkcije, a await se stavlja ispred poziva koji vraća Promise, pauzirajući izvršavanje dok se Promise ne razreši. Greške se hvataju standardnim try...catch blokom.

      Primer 4: Dohvatanje Podataka sa Javnog API-ja

      Cilj: Koristiti Fetch API i async/await sintaksu da se sa javnog API-ja (JSONPlaceholder) dohvati lista korisnika i prikaže na stranici.

      Lista korisnika sa API-ja

        HTML kod:

        <h1>Lista korisnika sa API-ja</h1>
        <ul id="api-user-list"></ul>

        JavaScript kod:

        const apiList = document.querySelector('#api-user-list');

        // Definišemo async funkciju
        async function fetchUsers() {
        try {
        // Čekamo da fetch završi i vrati odgovor
        const response = await fetch('https://jsonplaceholder.typicode.com/users');

        // Proveravamo da li je odgovor uspešan (status kod 200-299)
        if (!response.ok) {
        throw new Error(`HTTP greška! Status: ${response.status}`);
        }

        // Čekamo da se JSON podaci parsiraju
        const users = await response.json();

        // Prikazujemo podatke u DOM-u
        users.forEach(user => {
        const li = document.createElement('li');
        li.textContent = user.name;
        apiList.appendChild(li);
        });

        } catch (error) {
        // Hvatamo greške (mrežne ili iz našeg koda)
        console.error('Došlo je do greške:', error);
        apiList.textContent = 'Nije moguće učitati korisnike.';
        }
        }

        // Pozivamo funkciju da se izvrši
        fetchUsers();

        Analiza Koda:

        • Funkcija fetchUsers je označena kao async.
        • await fetch(...) pauzira funkciju dok se ne dobije odgovor od servera.
        • if (!response.ok) je važna provera, jer fetch ne odbacuje Promise za HTTP greške poput 404 ili 500.
        • await response.json() pauzira funkciju dok se telo odgovora ne pročita i parsira.
        • try...catch blok se koristi za elegantno hvatanje i obradu bilo kakvih grešaka koje se mogu desiti tokom procesa.
        • Na kraju, podaci se prikazuju na stranici koristeći standardnu DOM manipulaciju.

        Deo 5: Sinergija HTML-a i CSS-a (Izgradnja Statičkih Stranica)

        Primer 5: Interaktivna Forma sa Validacijom u Realnom Vremenu

        Cilj: Napraviti formu za prijavu koja proverava unos korisnika dok on kuca i pruža trenutnu povratnu informaciju o ispravnosti unosa, poboljšavajući korisničko iskustvo.

        .form-control.error input { border-color: red; }
        .form-control.success input { border-color: green; }
        .error-message {
        color: red;
        font-size: 0.8em;
        display: block;
        height: 1em; /* Rezerviše prostor da ne bi bilo skakanja layout-a */
        }

        const form = document.getElementById('validation-form');
        const email = document.getElementById('email');
        const password = document.getElementById('password');

        function showError(input, message) { ... }
        function showSuccess(input) { ... }

        function validateEmail(input) { ... }
        function validatePassword(input) { ... }

        email.addEventListener('input', () => validateEmail(email));
        password.addEventListener('input', () => validatePassword(password));

        form.addEventListener('submit', function(e) {

        Primer 6: Izgradnja Kompletne To-Do Liste sa Local Storage

        Cilj: Nadograditi prethodni primer "Shopping List" tako da se zadaci čuvaju u localStorage browsera, što znači da će ostati sačuvani čak i nakon što korisnik osveži stranicu.

        Moja To-Do Lista

          HTML kod:

          <div class="todo-container">
          <h1>Moja To-Do Lista</h1>
          <div class="input-group">
          <input type="text" id="task-input" placeholder="Dodaj novi zadatak...">
          <button id="add-task-btn">Dodaj</button>
          </div>
          <ul id="task-list"></ul>
          </div>

          JavaScript kod:

          const taskInput = document.getElementById('task-input');
          const addTaskBtn = document.getElementById('add-task-btn');
          const taskList = document.getElementById('task-list');

          // Učitavanje zadataka iz localStorage-a ili inicijalizacija praznog niza
          let tasks = JSON.parse(localStorage.getItem('tasks')) || [];

          function saveTasks() {
          localStorage.setItem('tasks', JSON.stringify(tasks));
          }

          function renderTasks() {
          taskList.innerHTML = '';
          tasks.forEach((task, index) => {
          const li = document.createElement('li');
          li.innerHTML = `<span class="${task.completed ? 'completed' : ''}">${task.text}</span><button class="delete-btn">Obriši</button>`;
          li.querySelector('span').addEventListener('click', () => toggleComplete(index));
          li.querySelector('.delete-btn').addEventListener('click', () => deleteTask(index));
          taskList.appendChild(li);
          });
          }

          function addTask() {
          const taskText = taskInput.value.trim();
          if (taskText !== '') {
          tasks.push({ text: taskText, completed: false });
          taskInput.value = '';
          saveTasks();
          renderTasks();
          }
          }

          function deleteTask(index) {
          tasks.splice(index, 1);
          saveTasks();
          renderTasks();
          }

          function toggleComplete(index) {
          tasks[index].completed = !tasks[index].completed;
          saveTasks();
          renderTasks();
          }

          addTaskBtn.addEventListener('click', addTask);
          taskInput.addEventListener('keypress', e => { if (e.key === 'Enter') addTask(); });
          renderTasks();

          Analiza Koda:

          • Struktura podataka: Zadaci se čuvaju kao niz objekata, gde svaki objekat ima text i completed status.
          • localStorage:
            • localStorage.setItem('tasks', JSON.stringify(tasks)): Pre čuvanja, niz objekata se mora konvertovati u JSON string, jer localStorage može da čuva samo stringove.
            • JSON.parse(localStorage.getItem('tasks')): Prilikom učitavanja, JSON string se parsira nazad u JavaScript niz objekata.
          • Jedinstveni izvor istine (Single Source of Truth): Sva logika (dodavanje, brisanje, izmena) prvo manipuliše tasks nizom. Nakon svake promene, pozivaju se saveTasks() i renderTasks(). Ovo osigurava da su podaci u localStorage i prikaz na stranici uvek sinhronizovani.
