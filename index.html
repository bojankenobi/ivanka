1. Uvod u Objektno Orijentisano Programiranje (OOP)

Dobrodošli na predavanje o Objektno Orijentisanom Programiranju! Danas ćemo zakoračiti u svet modernog programiranja i naučiti kako da razmišljamo o problemima na nov, intuitivniji način. Ovaj pristup nam pomaže da pišemo organizovaniji, fleksibilniji i lakši za održavanje kod, što je ključno za razvoj kompleksnih softverskih sistema.

Glavni cilj OOP-a je upravljanje složenošću (complexity). Kako softver raste, postaje sve teže pratiti sve delove i njihove međusobne veze. OOP nam daje alate da grupišemo povezane podatke i funkcionalnosti u samostalne jedinice (objekte), što drastično smanjuje mentalno opterećenje i omogućava timovima da efikasnije sarađuju.

Proceduralno Programiranje

Zamislite da pravite sendvič prateći recept korak po korak. To je suština proceduralnog programiranja. Pišemo niz instrukcija (procedura ili funkcija) koje računar izvršava redom. Fokus je na akcijama.

  • Fokus na procedurama i funkcijama.
  • Podaci i funkcije su odvojeni.
  • Teže za održavanje kod velikih projekata.

Objektno Orijentisano Programiranje (OOP)

Sada zamislite da gradite kuću od LEGO kockica. Svaka kockica je "objekat" sa svojim osobinama (boja, veličina) i mogućnostima (može se spojiti sa drugom). U OOP-u, svet posmatramo kroz objekte koji međusobno komuniciraju. Fokus je na stvarima (objektima).

  • Fokus na objektima koji objedinjuju podatke i ponašanje.
  • Kod je modularan i lakši za ponovnu upotrebu.
  • Intuitivnije preslikava stvarni svet u program.

2. Stubovi OOP-a: Pregled

Objektno orijentisano programiranje se zasniva na nekoliko ključnih principa koji ga čine moćnim. Mi ćemo ih zvati "stubovima" OOP-a. Razumevanje ovih koncepata je temelj za efikasno korišćenje OOP paradigme. Oni nam omogućavaju da gradimo softver koji je robustan, prilagodljiv i lak za razumevanje.

Klase i Objekti: Nacrt i Kuća

Ovo je najfundamentalniji koncept u OOP-u. Sve počinje odavde. Moramo jasno razlikovati šta je plan (klasa), a šta je stvarna stvar napravljena po tom planu (objekat).

Klasa (Class)

Klasa je nacrt ili šablon za kreiranje objekata. Ona definiše:

  • Atribute (Attributes): Osobine koje će objekat imati (npr. boja, ime, brzina). To su promenljive unutar klase.
  • Metode (Methods): Akcije koje će objekat moći da izvršava (npr. vozi, koči, laj). To su funkcije unutar klase.

Na primer, klasa `Automobil` bi definisala da svaki automobil ima boju, marku i brzinu, i da može da ubrza i zakoči.

Objekat (Object)

Objekat je instanca (konkretan primerak) klase. To je stvarna "stvar" kreirana prema nacrtu klase, sa konkretnim vrednostima za svoje atribute.

  • Moj plavi "Audi" je jedan objekat.
  • Tvoj crveni "BMW" je drugi objekat.
  • Oba su objekti klase `Automobil`.

Objekat je ono što "živi" u memoriji računara i sa čime radimo u programu.

Primer u Javi

Hajde da vidimo kako izgleda jednostavna klasa `Ucenik` u Javi.

public class Ucenik { // Atributi String ime; String prezime; int razred; // Metoda public void predstaviSe() { System.out.println("Zdravo, ja sam " + ime + " " + prezime + " i idem u " + razred + ". razred."); } }

Sada, hajde da kreiramo dva objekta (instance) te klase:

// Kreiranje prvog objekta Ucenik ucenik1 = new Ucenik(); ucenik1.ime = "Petar"; ucenik1.prezime = "Petrović"; ucenik1.razred = 3; // Kreiranje drugog objekta Ucenik ucenik2 = new Ucenik(); ucenik2.ime = "Jovana"; ucenik2.prezime = "Jovanović"; ucenik2.razred = 3; // Pozivanje metode nad objektima ucenik1.predstaviSe(); ucenik2.predstaviSe();

Konstruktori i `this` ključna reč

Kada kreiramo objekat sa `new Ucenik()`, mi zapravo pozivamo specijalnu metodu koja se zove konstruktor. Njegov zadatak je da inicijalizuje početno stanje objekta.

  • Svaka klasa ima podrazumevani (default) konstruktor ako ga mi ne definišemo.
  • Možemo definisati i sopstvene konstruktore da bismo primili početne vrednosti za atribute.
  • Ključna reč `this` se odnosi na trenutni objekat. Koristi se za razlikovanje atributa klase od parametara metode (ili konstruktora) kada imaju isto ime.
public class Ucenik { String ime; String prezime; int razred; // Naš konstruktor koji prihvata početne vrednosti public Ucenik(String ime, String prezime, int razred) { // 'this.ime' je atribut klase // 'ime' je parametar prosleđen konstruktoru this.ime = ime; this.prezime = prezime; this.razred = razred; } public void predstaviSe() { System.out.println("Zdravo, ja sam " + this.ime + " " + this.prezime); } } // Sada je kreiranje objekata mnogo čistije: Ucenik ucenik1 = new Ucenik("Petar", "Petrović", 3); Ucenik ucenik2 = new Ucenik("Jovana", "Jovanović", 3); ucenik1.predstaviSe();

Enkapsulacija: Sigurnosna Kutija

Enkapsulacija je princip "pakovanja" podataka (atributa) i metoda koje rade sa tim podacima u jednu celinu - objekat. Ali to nije sve! Ključni deo je sakrivanje informacija (information hiding).

Zamislite bankomat. Vi možete da ubacite karticu, unesete PIN i podignete novac. Ne znate (i ne treba da znate) kako bankomat interno proverava stanje na računu ili broji novčanice. On vam nudi jednostavan interfejs, a sakriva unutrašnju složenost. To je enkapsulacija.

Kako to radi?

  • Atributi klase se deklarišu kao private. To znači da im se ne može direktno pristupiti van klase.
  • Kreiraju se javne metode, tzv. geteri i seteri (public), koje služe kao kontrolisani pristup privatnim atributima.

Zašto je ovo važno?

  • Kontrola: Seteri nam omogućavaju da dodamo logiku za proveru pre nego što promenimo vrednost atributa (npr. ocena ne može biti veća od 5).
  • Sigurnost: Sprečava se slučajna ili namerna izmena stanja objekta na nepredviđen način.
  • Fleksibilnost: Možemo promeniti unutrašnju implementaciju klase bez da utičemo na kod koji je koristi, sve dok javni interfejs (metode) ostaje isti.
  • Održivost: Kada je unutrašnje stanje objekta zaštićeno, lakše je pronaći i ispraviti greške, jer znamo da se stanje moglo promeniti samo kroz definisane setere.

Primer: Klasa `Ucenik` sa enkapsulacijom

public class Ucenik { // Atributi su sada private! private String ime; private int ocena; // Javni seter za 'ime' public void setIme(String novoIme) { this.ime = novoIme; } // Javni geter za 'ime' public String getIme() { return this.ime; } // Seter za 'ocena' sa logikom za proveru public void setOcena(int novaOcena) { if (novaOcena >= 1 && novaOcena <= 5) { this.ocena = novaOcena; } else { System.out.println("Greška: Ocena mora biti između 1 i 5."); } } // Geter za 'ocena' public int getOcena() { return this.ocena; } }

Sada, umesto direktnog pristupa `ucenik1.ocena = 10;`, moramo koristiti metodu:

Ucenik ucenik1 = new Ucenik(); ucenik1.setIme("Marko"); ucenik1.setOcena(5); // Ovo je u redu ucenik1.setOcena(7); // Ovo će ispisati grešku! System.out.println("Učenik: " + ucenik1.getIme() + ", Ocena: " + ucenik1.getOcena());

Nasleđivanje: Porodično Stablo

Nasleđivanje (Inheritance) je mehanizam koji omogućava jednoj klasi da "pokupi" (nasledi) atribute i metode od druge klase. Ovo je jedan od najmoćnijih načina za ponovnu upotrebu koda u OOP-u.

Ključni termini

  • Nadklasa (Superclass): Klasa od koje se nasleđuje. Naziva se i roditeljska (parent) ili osnovna (base) klasa.
  • Podklasa (Subclass): Klasa koja nasleđuje. Naziva se i dečija (child) ili izvedena (derived) klasa.

Koristimo ključnu reč extends da bismo ostvarili nasleđivanje.

Primer iz života

Imamo opšti koncept `Vozilo`. Svako vozilo ima brzinu i može da se kreće.

`Automobil` je specifična vrsta vozila. On nasleđuje sve od `Vozila` (ima brzinu, kreće se), ali dodaje i nešto svoje, npr. broj vrata.

`Bicikl` je takođe vrsta vozila. I on nasleđuje od `Vozila`, ali ima svoj specifičan atribut, npr. broj brzina.

"IS-A" Odnos (JESTE)

Nasleđivanje modelira "IS-A" (jeste) odnos. Ovo je ključno za pravilno korišćenje. Pre nego što upotrebite nasleđivanje, postavite sebi pitanje: Da li je podklasa zaista vrsta nadklase?
✔ `Pas` jeste `Životinja`. (Dobro)
✔ `Automobil` jeste `Vozilo`. (Dobro)
❌ `Automobil` nije `Točak`. (Loše - ovo bi bio "HAS-A" odnos)

Primer u Javi

Prvo, osnovna (roditeljska) klasa:

public class Zivotinja { String ime; public void jedi() { System.out.println("Ova životinja jede."); } }

Sada, izvedena (dečija) klasa koja nasleđuje `Zivotinju`:

// Pas JE Životinja public class Pas extends Zivotinja { public void laj() { System.out.println("Av av!"); } } // Macka JE Životinja public class Macka extends Zivotinja { public void mjauci() { System.out.println("Mjau!"); } }

Kako se koristi:

Pas mojPas = new Pas(); mojPas.ime = "Lesi"; // 'ime' je nasleđeno iz klase Zivotinja mojPas.jedi(); // 'jedi()' je nasleđeno iz klase Zivotinja mojPas.laj(); // 'laj()' je specifično za klasu Pas

Ključna reč `super`

Ključna reč `super` se koristi unutar podklase da bi se pozvale metode ili konstruktori iz nadklase. Ovo je korisno kada želimo da proširimo, a ne da potpuno zamenimo, ponašanje nadklase.

public class Osoba { String ime; public Osoba(String ime) { this.ime = ime; System.out.println("Kreirana je Osoba."); } public void predstaviSe() { System.out.println("Ja sam osoba, zovem se " + ime); } } public class Student extends Osoba { String fakultet; public Student(String ime, String fakultet) { // 1. Pozivamo konstruktor nadklase (Osoba) super(ime); this.fakultet = fakultet; System.out.println("Kreiran je Student."); } @Override public void predstaviSe() { // 2. Pozivamo metodu predstaviSe() iz nadklase super.predstaviSe(); System.out.println("Studiram na " + fakultet); } } // Upotreba: Student s = new Student("Milan", "ETF"); s.predstaviSe();

Polimorfizam: Više Oblika

Polimorfizam na grčkom znači "imati više oblika". U Javi, polimorfizam nam omogućava da izvršimo istu akciju na različite načine. Postoje dva glavna tipa polimorfizma:

1. Statički (Compile-time) Polimorfizam: Preopterećenje Metoda (Method Overloading)

Ovo se dešava kada u istoj klasi imamo više metoda sa istim imenom, ali različitim parametrima (različit broj parametara, različit tip, ili oboje). Kompajler odlučuje koju metodu da pozove na osnovu argumenata koje smo prosledili.

class Kalkulator { public int saberi(int a, int b) { return a + b; } // Ista metoda, ali sa 3 parametra public int saberi(int a, int b, int c) { return a + b + c; } // Ista metoda, ali sa double tipom public double saberi(double a, double b) { return a + b; } } Kalkulator k = new Kalkulator(); System.out.println(k.saberi(2, 3)); // Poziva prvu metodu System.out.println(k.saberi(2, 3, 4)); // Poziva drugu metodu System.out.println(k.saberi(2.5, 3.5)); // Poziva treću metodu

2. Dinamički (Run-time) Polimorfizam: Redefinisanje Metoda (Method Overriding)

Ovo se dešava kada podklasa pruži specifičnu implementaciju metode koja je već definisana u njenoj nadklasi. Odluka o tome koja metoda će se izvršiti donosi se u vreme izvršavanja programa (run-time), na osnovu tipa objekta.

Ovo je ono što najčešće podrazumevamo pod "polimorfizmom" u OOP kontekstu. Ključno je za fleksibilnost i nadogradivost koda.

class Zivotinja { public void oglasiSe() { System.out.println("Životinja se oglašava"); } } class Pas extends Zivotinja { @Override public void oglasiSe() { System.out.println("Pas laje: Av av!"); } } // Upotreba Zivotinja z = new Pas(); // z je tipa Zivotinja, ali objekat je Pas z.oglasiSe(); // U vreme izvršavanja, JVM vidi da je objekat Pas // i poziva metodu iz klase Pas.

Apstrakcija: Fokusiranje na Bitno

Apstrakcija je proces sakrivanja implementacionih detalja i prikazivanja samo funkcionalnosti korisniku. Drugim rečima, korisnik zna šta objekat radi, ali ne i kako to radi.

Dobar primer je vožnja automobila. Da biste vozili, morate znati da koristite volan, pedale i menjač (interfejs). Ne morate znati kako motor sa unutrašnjim sagorevanjem radi, kako se snaga prenosi na točkove, ili kako radi elektronski sistem ubrizgavanja goriva (implementacija).

Kako se postiže u Javi?

Apstrakcija se u Javi najčešće postiže korišćenjem:

  • Apstraktnih klasa (Abstract Classes): Klasa koja se ne može instancirati (ne može se kreirati objekat od nje). Može imati i apstraktne (bez implementacije) i konkretne metode. Služi kao osnova za druge klase koje je nasleđuju.
  • Interfejsa (Interfaces): Potpuno apstraktan "ugovor" koji definiše skup metoda koje klasa mora da implementira. Ne može imati implementaciju metoda (u starijim verzijama Jave).

Zašto je ovo važno?

  • Smanjuje složenost: Fokusiramo se na interakciju sa objektom na višem nivou.
  • Olakšava održavanje: Možemo promeniti internu implementaciju bez promene spoljnog koda koji je koristi.
  • Povećava sigurnost: Sprečava korisnika da napravi promene koje bi mogle da naruše konzistentnost objekta.

Primer: Apstraktna klasa `Oblik`

Definišemo apstraktnu klasu `Oblik`. Svaki oblik mora imati mogućnost da izračuna svoju površinu, ali ne znamo kako se to radi za "apstraktni" oblik. Zato je metoda `povrsina()` apstraktna, i svaka konkretna klasa koja nasledi `Oblik` mora da pruži svoju implementaciju.

// Apstraktna klasa - ne može se kreirati objekat tipa Oblik abstract class Oblik { // Apstraktna metoda - nema telo, samo potpis // Svaka podklasa MORA da je implementira public abstract double povrsina(); // Apstraktna klasa može imati i konkretne metode public void prikazi() { System.out.println("Ovo je neki geometrijski oblik."); } } class Krug extends Oblik { private double poluprecnik = 5.0; // Pružamo implementaciju za apstraktnu metodu @Override public double povrsina() { return Math.PI * poluprecnik * poluprecnik; } } // Greška! Ne možemo kreirati objekat apstraktne klase. // Oblik o = new Oblik(); // Ovo je ispravno Oblik krug = new Krug(); krug.prikazi(); System.out.println("Površina kruga je: " + krug.povrsina());

3. UML: Jezik za Crtanje Softvera

Unified Modeling Language (UML) je standardizovani grafički jezik za vizuelizaciju, specificiranje, konstruisanje i dokumentovanje softverskih sistema. To je kao arhitektonski nacrt za zgradu, samo što je za softver. Pomaže nam da planiramo i komuniciramo ideje pre nego što napišemo kod.

Postoji mnogo vrsta UML dijagrama, ali za nas je na početku najvažniji Dijagram Klasa (Class Diagram), jer direktno prikazuje strukturu našeg OOP koda.

Dijagram Klasa

Dijagram klasa prikazuje klase u sistemu, njihove atribute, metode i odnose između klasa. Osnovni element je pravougaonik podeljen na tri dela:

ImeKlase

- atribut1: TipPodatka

+ atribut2: TipPodatka

+ metoda1(parametar: Tip): PovratniTip

- metoda2(): void

  • Gornji deo: Ime klase.
  • Srednji deo: Lista atributa.
  • Donji deo: Lista metoda.

Oznake vidljivosti (Visibility):

  • + public: Vidljivo svima.
  • - private: Vidljivo samo unutar klase.
  • # protected: Vidljivo unutar klase i u njenim podklasama.

Primer: UML dijagram za klasu `Ucenik`

Ovako bi izgledao UML dijagram za našu `Ucenik` klasu sa enkapsulacijom.

Ucenik

- ime: String

- ocena: int

+ setIme(novoIme: String): void

+ getIme(): String

+ setOcena(novaOcena: int): void

+ getOcena(): int

Ovaj dijagram jasno i sažeto prikazuje sve bitne informacije o klasi `Ucenik`.

Odnosi Između Klasa

Osim nasleđivanja, klase mogu biti u različitim odnosima. UML nam pomaže da vizuelizujemo te veze.

Asocijacija

"koristi"

Klasa A ⎯⎯⎯⎯⎯ Klasa B

Student koristi Knjigu. Nezavisni su.

Agregacija

"ima"

Klasa A ◇⎯⎯⎯⎯⎯ Klasa B

Departman ima Profesora. Profesor može postojati i bez departmana.

Kompozicija

"je deo"

Klasa A ◆⎯⎯⎯⎯⎯ Klasa B

Kuća se sastoji od Sobe. Soba ne može postojati bez kuće.

4. Praktični Primeri

Teorija je važna, ali najbolje se uči kroz praksu. Hajde da sada primenimo sve što smo naučili na konkretnim primerima. Možete interagovati sa kodom i videti kako promene utiču na rezultat.

Primer 1: Klasa `Automobil`

Kreirajmo jednostavnu klasu `Automobil` sa atributima za marku i brzinu, i metodama za ubrzavanje i prikaz informacija. Pritisnite dugme "Pokreni" da kreirate objekat i pozovete njegove metode.

Java Kod:

public class Automobil { public String marka; public int brzina; public Automobil(String marka) { this.marka = marka; this.brzina = 0; } public void ubrzaj(int kolicina) { this.brzina += kolicina; } public String prikaziInfo() { return "Automobil marke " + marka + " se kreće brzinom od " + brzina + " km/h."; } } // Glavni program Automobil mojAuto = new Automobil("Audi"); mojAuto.ubrzaj(50); System.out.println(mojAuto.prikaziInfo()); mojAuto.ubrzaj(30); System.out.println(mojAuto.prikaziInfo());

Primer 2: Bankovni Račun

Ovaj primer demonstrira enkapsulaciju. Stanje računa je privatno i može se menjati samo preko javnih metoda `uplati` i `podigni`, koje sadrže logiku za proveru.

Java Kod:

public class BankovniRacun { private double stanje; public BankovniRacun(double pocetnoStanje) { this.stanje = pocetnoStanje; } public double getStanje() { return this.stanje; } public void uplati(double iznos) { if (iznos > 0) { this.stanje += iznos; } } public void podigni(double iznos) { if (iznos > 0 && iznos <= this.stanje) { this.stanje -= iznos; } else { System.out.println("Nemate dovoljno sredstava."); } } } // Glavni program BankovniRacun racun = new BankovniRacun(1000); racun.uplati(500); racun.podigni(200); racun.podigni(1500); // Ovo će prijaviti grešku System.out.println("Trenutno stanje: " + racun.getStanje());
